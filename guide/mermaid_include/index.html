    Mermaid 图表引用¶

    本文档介绍如何在 Markdown 文件中引用外部 .mmd 文件，使其在 MkDocs 编译后以图形方式渲染。

    原理¶

    通过 pymdownx.snippets + pymdownx.superfences 组合实现：

    1. pymdownx.snippets（预处理器）在编译早期将 --8<-- "filename" 替换为文件内容
    2. pymdownx.superfences 随后将包含 mermaid 内容的代码块渲染为 <pre class="mermaid">，由浏览器端 Mermaid.js 绘制图形

    基本用法¶

    在 Markdown 文件中使用以下语法：

    ```mermaid
%%{init: {'theme': 'base', 'themeVariables': {'primaryColor': '#e8f5e9', 'primaryBorderColor': '#2e7d32', 'primaryTextColor': '#1b5e20', 'lineColor': '#43a047', 'secondaryColor': '#fff3e0', 'tertiaryColor': '#e3f2fd'}}}%%

graph TB
    subgraph APP["应用层 Application Layer"]
        direction LR
        API["elog_api.h<br/>elog_debug/elog_info/elog_warn/elog_error<br/>elog_event_begin/end<br/>elog_assert"]
        MACRO["ELOG_D / ELOG_I / ELOG_W / ELOG_E<br/>ELOG_TAG_D(tag, fmt, ...)<br/>ELOG_EVENT(event_id, ...)"]
        API --- MACRO
    end

    subgraph CORE["核心层 Core Layer"]
        direction TB
        MSG["LogMessage<br/>─────────────────<br/>log_id: uint8_t<br/>level: uint8_t<br/>timestamp: uint32_t<br/>tag: const char*<br/>file: const char*<br/>line: uint16_t<br/>pid: uint16_t<br/>tid: uint16_t<br/>msg: char[]"]

        DISPATCH["Dispatcher<br/>─────────────────<br/>write_dispatcher()<br/>─────────────────<br/>logger_function_t*<br/>注册/替换日志后端"]

        FILTER["LogFilter<br/>─────────────────<br/>global_level: uint8_t<br/>tag_level_map[]<br/>module_level_map[]<br/>─────────────────<br/>should_log()<br/>set_level()"]

        FMT["Formatter<br/>─────────────────<br/>format_text()<br/>format_binary()<br/>─────────────────<br/>时间戳格式化<br/>彩色输出<br/>源码位置"]

        DISPATCH --- MSG
        MSG --- FILTER
        FILTER --- FMT
    end

    subgraph BUFFER["缓冲层 Buffer Layer"]
        direction TB
        BUF_IFACE["LogBuffer (抽象接口)<br/>─────────────────<br/>log()<br/>flush()<br/>clear()<br/>size() / capacity()"]

        RING_BUF["RingLogBuffer<br/>─────────────────<br/>ring_buffer_t<br/>动态/静态分配<br/>无锁 SPSC<br/>自动覆写/阻塞"]

        SHM_BUF["ShmLogBuffer<br/>─────────────────<br/>共享内存区域<br/>多进程安全<br/>POSIX shm"]

        BUF_IFACE --- RING_BUF
        BUF_IFACE --- SHM_BUF
    end

    subgraph TRANSPORT["传输层 Transport Layer"]
        direction LR
        T_IFACE["LogTransport (抽象接口)<br/>─────────────────<br/>open() / close()<br/>write(entry) / flush()"]

        T_UART["UartTransport<br/>─────────────────<br/>UART / serial<br/>DMA 零拷贝<br/>波特率自适应"]

        T_FILE["FileTransport<br/>─────────────────<br/>文件系统<br/>滚动日志<br/>可选压缩"]

        T_NET["NetTransport<br/>─────────────────<br/>UDP/TCP<br/>远程 syslog<br/>TLS 可选"]

        T_CAN["CanTransport<br/>─────────────────<br/>CAN 总线<br/>UDS 诊断<br/>ISO-TP"]

        T_RTOS["RtosQueueTransport<br/>─────────────────<br/>RTOS 消息队列<br/>FreeRTOS/CMSIS-RTOS<br/>线程间传递"]

        T_IFACE --- T_UART
        T_IFACE --- T_FILE
        T_IFACE --- T_NET
        T_IFACE --- T_CAN
        T_IFACE --- T_RTOS
    end

    subgraph BACKEND["后端服务层 Backend Service Layer"]
        direction LR
        DAEMON["elogd 守护进程<br/>─────────────────<br/>多源汇聚<br/>优先级裁剪<br/>统一输出"]

        LOGCAT["elogcat CLI<br/>─────────────────<br/>多缓冲区读取<br/>过滤/着色/格式化"]

        COLLECTOR["日志采集器<br/>─────────────────<br/>远程上报<br/>OTA 配套"]
    end

    subgraph STORAGE["持久化层 Storage Layer"]
        direction LR
        FS["文件系统<br/>─────────────────<br/>FAT32 / LittleFS<br/>滚动策略<br/>可选压缩"]

        PSTORE["pstore / NVRAM<br/>─────────────────<br/>掉电保持<br/>关键日志<br/>崩溃恢复"]

        FLASH["Flash 环形区<br/>─────────────────<br/>原始分区<br/>磨损均衡<br/>断电安全"]
    end

    subgraph CONFIG["配置层 Config Layer"]
        direction LR
        COMPILE["编译期配置<br/>─────────────────<br/>ELOG_MAX_TAG_LEN<br/>ELOG_MAX_MSG_LEN<br/>ELOG_BUFFER_SIZE<br/>ELOG_LEVEL_DEFAULT<br/>ELOG_COLOR_ENABLE"]

        RUNTIME["运行时配置<br/>─────────────────<br/>动态级别调整<br/>模块开关<br/>传输目标切换<br/>缓冲区大小调整"]

        COMPILE --- RUNTIME
    end

    APP -->|调用| CORE
    CORE -->|写入| BUFFER
    BUFFER -->|分发| TRANSPORT
    TRANSPORT -->|输出| STORAGE
    TRANSPORT -->|网络| BACKEND
    BACKEND -->|采集| STORAGE
    CONFIG -.->|编译期宏| CORE
    CONFIG -.->|编译期宏| BUFFER
    CONFIG -.->|运行时 API| CORE

    style APP fill:#e8f5e9,stroke:#2e7d32,stroke-width:2px

```

    示例：引用其他目录的 .mmd 文件¶

    ```mermaid
%%{init: {'theme': 'base', 'themeVariables': {'primaryColor': '#e8f5e9', 'primaryBorderColor': '#2e7d32', 'lineColor': '#43a047'}}}%%

graph LR
    subgraph Sources["日志源 (Log Sources)"]
        APP["App A<br/>elog_init('app_a')"]
        DRV["Driver<br/>elog_init('driver')"]
        MIDL["Middleware<br/>elog_init('middleware')"]
        KERN["Kernel Module<br/>klog_write()"]
        RTOS["RTOS Event<br/>elog_event()"]
    end

    subgraph ClientLib["客户端库 (libelog.so)"]
        API["elog API<br/>elog_d/i/w/e()"]
        EVENT["Event API<br/>elog_event_begin/end()"]
        WRITE["Transport Registry<br/>dispatch()"]
        COMPRESS["Compress (可选)<br/>lz4 / miniz"]
        WRITE_TX["writer_func_t*<br/>函数指针分发"]

        API --> EVENT
        API --> WRITE
        EVENT --> WRITE
        WRITE --> WRITE_TX
    end

    subgraph Daemon["elogd 守护进程"]
        IN_LOG["LogListener<br/>/var/run/elogdw<br/>SOCK_DGRAM"]
        IN_CMD["CmdListener<br/>/var/run/elogd<br/>SOCK_STREAM"]
        OUT_READ["LogReader<br/>/var/run/elogdr<br/>SOCK_SEQPACKET"]
        BUF["RingLogBuffer<br/>×log_id"]
        STATS["LogStatistics"]
        PRUNE["LogPrune"]
        TAG["EventTagMap"]

        IN_LOG --> BUF
        BUF --> STATS
        BUF --> PRUNE
        TAG -.->|event 解码| OUT_READ
        IN_CMD -->|命令| BUF
        IN_CMD --> STATS
        IN_CMD --> PRUNE
        BUF --> OUT_READ
    end

    subgraph Outputs["输出目标 (Log Outputs)"]
        UART_OUT["UART Console<br/>实时调试"]
        FILE_OUT["Log Files<br/>/var/log/elog/<br/>滚动+压缩"]
        NET_OUT["Network<br/>UDP syslog<br/>TCP 长连接"]
        FLASH_OUT["Flash Ring<br/>掉电保持<br/>pstore 分区"]
    end

    subgraph Tools["工具链"]
        CAT["elogcat<br/>多缓冲区读取<br/>过滤/着色"]
        DUMP["elogdump<br/>二进制转文本<br/>离线分析"]
        CONF["elogctl<br/>运行时配置<br/>级别/裁剪规则"]
    end

    APP --> API
    DRV --> API
    MIDL --> API
    KERN -->|"procfs/ioctl"| IN_LOG
    RTOS --> API

    WRITE -->|DGRAM| IN_LOG
    WRITE -->|"可选"| COMPRESS
    COMPRESS -->|pstore| FLASH_OUT

    OUT_READ -->|"PUSH"| CAT
    OUT_READ -->|"PUSH"| NET_OUT
    OUT_READ -->|"PUSH"| UART_OUT
    BUF -->|"定时 flush"| FILE_OUT

    CONF -->|"SOCK_STREAM"| IN_CMD

    style Sources fill:#e8f5e9,stroke:#2e7d32,stroke-width:2px
    style ClientLib fill:#fff8e1,stroke:#f57f17,stroke-width:2px
    style Daemon fill:#e3f2fd,stroke:#1565c0,stroke-width:2px
    style Outputs fill:#fce4ec,stroke:#c62828,stroke-width:2px
    style Tools fill:#f3e5f5,stroke:#6a1b9a,stroke-width:2px

```

    .mmd 文件编写规范¶

    • 使用标准 Mermaid 语法（graph、sequenceDiagram、classDiagram 等）
    • 文件编码为 UTF-8
    • 文件内不要包含 ``` 围栏标记，只需写 Mermaid 图表内容本身
    graph LR
    A[编写 .mmd 文件] --> B[在 md 中用 --8<-- 引用]
    B --> C[mkdocs build]
    C --> D[浏览器中查看图形]

    配置参考¶

    mkdocs.yml 中的相关配置：

    markdown_extensions:
  - pymdownx.snippets:
      base_path:
        - docs    # snippets 搜索根目录
      check_paths: true   # 文件不存在时报错
  - pymdownx.superfences:
      custom_fences:
        - name: mermaid
          class: mermaid
          format: !!python/name:pymdownx.superfences.fence_code_format

    常见问题¶

    Q: 构建报错 File not found

    检查文件路径是否相对于 base_path 正确。如果 .mmd 文件在 docs/src/packages/embeded_logging/ 下，路径应为 src/packages/embeded_logging/xxx.mmd。

    Q: 图表不渲染，只显示代码文本

    确认浏览器控制台是否有 Mermaid.js 加载错误。MkDocs Material 主题内置了 Mermaid 支持（9.0+），确保主题版本 >= 9.0。
