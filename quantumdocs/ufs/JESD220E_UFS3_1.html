JESD220E UFS 3.1 规范深度分析

本文档基于 JEDEC Universal Flash Storage (UFS) Version 3.1 规范 (JESD220E, January 2020), 系统梳理 UFS 3.1 的架构、协议、电气特性和功能特性。

参考文档：docs/pdf/UFS/JESD220E_UFS3_1.md

概述

UFS 简介 [Section 4]_

Universal Flash Storage (UFS) 是一种简单、高性能的串行接口大容量存储设备，主要用于移动系统中主机处理器和大容量存储设备之间。

目标应用 [Section 4.1]_:

  • 手机、UMPC(超移动 PC)、DSC(数字静物相机)、PMP(便携式媒体播放器)、MP3 播放器

  • 需要大容量存储、可启动大容量存储和外部卡的应用

目标设备类型 [Section 4.1]_:

  • 外部卡

  • 嵌入式设备 (大容量存储和可启动大容量存储)

  • 未来扩展的设备类型 (I/O 设备、摄像头、无线设备等)

拓扑结构 [Section 4.1]_:

  • 每个 UFS 端口连接一个设备

层叠架构 [Section 4.1, Figure 5.1]_:

UFS Application Layer (UAP)
├── UFS Command Set Layer (UCS)
│   ├── UFS Native Command Set
│   ├── Simplified SCSI Command Set
│   └── Future Extension
├── Device Manager (Query Request)
└── Task Manager

UFS Transport Protocol Layer (UTP)

UFS InterConnect Layer (UIC)
├── MIPI UniPro (数据链路层)
└── MIPI M-PHY (物理层)

通用特性 [Section 4.1]_

目标性能:

  • 支持高速 GEAR1/GEAR2/GEAR3/GEAR4 (全部强制支持)

接口特性 [Section 4.2]_:

  • 三种电源供电: - VCCQ: 1.2 V (标称) - VCCQ2: 1.8 V (标称) - VCC: 2.5 V/3.3 V (标称)

  • 信号定义遵循 [MIPI-M-PHY]: - 400 mVp (未端接) - 200 mVp (端接)

  • 8b10b 线路编码

  • 高可靠性 - BER 低于 10⁻¹⁰

  • 两种信令方案: - 低速模式：PWM 信令 - 高速突发模式

  • 低速和高速模式均定义多个 GEAR

  • Adapt (M-PHY 4.1): M-RX 均衡器训练以适应信道特性

功能特性 [Section 4.3]_

UFS 功能特性主要是 NAND 管理特性，包括:

  • 类似 eMMC 的功能特性

  • 启动操作模式

  • 多个可配置特性的逻辑单元

  • Replay Protected Memory Block (RPMB)

  • 可靠写入操作

  • 后台操作

  • 安全操作 (Purge 和 Erase 增强数据安全)

  • 写保护选项 (包括永久和上电写保护)

  • RPMB 的签名访问

  • 硬件复位信号

  • 任务管理操作

  • 电源管理操作

架构概述

顶层架构 [Section 5.1]_

UFS 通信采用分层通信架构，基于 SCSI SAM 架构模型 [SAM]。

应用层 [Section 5.1.1]_:

由 UFS 命令集 (UCS)、设备管理器和任务管理器组成:

  • UCS: 处理正常命令 (如读、写等),可支持多种命令集

  • Task Manager: 处理命令队列控制相关命令

  • Device Manager: 提供设备级控制 (如 Query Request) 和底层链路层控制

UFS 设备管理器 [Section 5.1.2]_:

两个主要职责:

  1. 处理设备级操作 (电源管理、数据传输设置、后台操作使能等)

  2. 管理设备级配置 (维护和存储描述符集)

服务访问点 [Section 5.1.3-5.1.5, Figure 5.2, 5.3]_:

设备管理器通过两个服务访问点与下层交互:

  • UDM_SAP (UFS Device Manager Service Access Point): - UTP 层暴露给设备管理器的 SAP - 用于处理设备级操作和配置 - 对应 Query Request 和 Query Response 功能 - 例如：处理描述符的 query request

  • UIO_SAP (UFS IC IO control Service Access Point): - UIC 层暴露给设备管理器的 SAP - 用于触发 UIC 层复位和传输 UIC 管理功能相关请求/响应 - 在 UniPro 中对应 DME_SAP - 提供原语包括:

    • DME_RESET: 复位整个 UniPro 协议栈

    • DME_ENDPOINTRESET: UFS 主机要求 UFS 设备执行复位

UFS 传输协议层 [Section 5.1.6]_:

UTP 层为上层提供服务，UPIU (UFS Protocol Information Unit) 在 UFS 主机和设备的 UTP 层之间交换。

UTP 层提供三个访问点:

  1. UDM_SAP: 执行设备级管理 (如描述符访问)

  2. UTP_CMD_SAP: 传输命令

  3. UTP_TM_SAP: 传输任务管理功能 (如"abortion task")

UFS 互连层 [Section 5.1.7]_:

最底层是 UFS Interconnect Layer (UIC),处理 UFS 主机和设备之间的连接。

UIC 组成: - MIPI UniPro - MIPI M-PHY

UIC 提供两个服务访问点: - UIC_SAP: 在 UFS 主机和设备之间传输 UPIU (对应 UniPro 中的 T_SAP) - UIO_SAP: 管理 UIC (对应 UniPro 中的 DME_SAP)

UFS 拓扑 [Section 5.1.8]_:

此版本标准假设只有一个设备连接到 UFS 端口，其他拓扑可能在将来版本中定义。

系统模型 [Section 5.2, Figure 5.4]_

UFS 系统示例展示 UFS 主机如何连接到 UFS 设备，以及 UFS 主机控制器的位置和相关 UFS HCI 接口。

UFS 主机组件:

  • Application: 希望与 UFS 设备通信的实体

  • UFS Driver: 通过 UFS HCI 管理 UFS 主机控制器

  • UFS Host Controller: 包含一组寄存器 (UFS HCI)

  • UFS Host Reg: 主机控制器暴露的寄存器组

UFS 设备组件:

  • Device Level Managing: 设备级管理 (如电源管理)

  • Descriptors: 存储配置相关信息

  • LU-0 to LU-N: 多个逻辑单元，执行读、写等功能

  • UIC: 包含 MIPI UniPro 和 MIPI M-PHY

UFS 接口信号:

  • REF_CLK: 参考时钟

  • RST_n: 复位信号

  • DOUT_t/c: 上行差分输出

  • DIN_t/c: 下行差分输入

系统启动和枚举 [Section 5.3]_

系统从可启动 UFS 设备启动在上电后开始，当 UFS InterConnect Layer (MIPI M-PHY 和 UniPro) 完成启动序列后。

启动流程:

  • 启动代码可从适当的 boot logical unit 读取

  • 或者根据需要，boot ROM 代码可在读取启动代码前重新配置 MIPI M-PHY 和 UniPro 到适当设置

  • 多个 boot logical unit 可能在 UFS 设备中可用，但上电时只有一个激活

  • 适当的描述符可用于配置启动过程

  • 启动期间，通过 SCSI 命令支持对 boot logical unit 的访问

SCSI 域模型 [Section 5.5.1, Figure 5.5, 5.6]_

CDB、状态、任务管理 [Section 5.5.1.2]_:

UTP 采用 Command Descriptor Block (CDB) 格式用于命令、设备状态数据层次结构和报告方法，以及未完成任务的任务管理功能，这些都在 [SAM] 中描述。

无论 UTP 传递的命令协议如何，UFS 设备应统一采用 SCSI CDB、状态和任务管理功能。

Nexus [Section 5.5.1.3]_:

Nexus 表示 Initiator、Target、Logical Unit 和 Command (Task) 之间的关系。

Nexus 表示法:

I_T_L_Q nexus
其中：I = Initiator, T = Target, L = Logical Unit, Q = Command

要求:

  • UFS 定义中应至少有一个 initiator device

  • 只能有一个 target device (UFS 设备)

  • UFS 设备中应有一个或多个 logical units

  • 命令标识符 (即 I_T_L_Q nexus 中的 Q) 由 Initiator device 分配，以在特定 I_T_L nexus 上下文中唯一标识一个命令

  • 允许同一时间有多个命令在 I_T_L nexus 中 outstanding

重叠命令:

当 task manager 或 task router 在命令完成其命令生命周期之前检测到 I_T_L_Q nexus 重复使用时，发生重叠命令。

UFS 中不允许并发重叠命令。每个命令应有唯一的 Task Tag。UFS 设备不需要检测重叠命令。

SCSI 命令模型 [Section 5.5.1.4]_:

所有命令请求源自 Initiator device 中的 application clients。

Application Client 使用以下过程调用请求命令处理:

Service Response = Execute Command (
    IN (I_T_L_Q Nexus, CDB, Task Attribute,
        [Data-In Buffer Size], [Data-Out Buffer], [Data-Out Buffer Size],
        [CRN], [Command Priority]),
    OUT ([Data-In Buffer], [Sense Data], [Sense Data Length],
         Status, [Status Qualifier])
)

UTP Command、Response、Ready-to-Transfer、Data-Out、Data-In UPIU 头中的参数字段包含 [SAM] 合规的 Execute Command 过程调用的输入和输出参数所需信息。

SCSI 任务管理功能 [Section 5.5.1.5]_:

Application client 使用以下过程调用请求任务管理功能处理:

Service Response = Function name (
    IN (Nexus),
    OUT ([Additional Response Information])
)

UTP Task Management Request 和 UTP Task Management Response 头中的参数字段包含 [SAM] 合规的任务管理功能过程调用的输入和输出参数所需信息。

应用和命令层 [Section 5.6]_

UFS 接口设计为协议无关接口，但 SCSI 被选为此标准的基础协议层。描述符可用于识别和选择 UFS 接口的适当协议。

命令集层的主要功能:

  • 建立 UFS 主机和设备之间的数据交换方法

  • 提供基础设备管理能力

UFS SCSI 命令集 [Section 5.6]:

  1. SBC 兼容命令 [SBC]: - FORMAT UNIT - READ (6) 和 READ (10) - READ CAPACITY (10) - REQUEST SENSE - SEND DIAGNOSTIC - UNMAP - WRITE (6) 和 WRITE (10)

  2. SPC 兼容命令 [SPC]: - INQUIRY - REPORT LUNS - READ BUFFER - TEST UNIT READY - WRITE BUFFER - SECURITY PROTOCOL IN 和 SECURITY PROTOCOL OUT

  3. **UFS 应用的 SCSI 操作命令**(与现有 SCSI 驱动兼容): - MODE SELECT (10) 和 MODE SENSE (10) - PRE-FETCH (10) - START STOP UNIT - SYNCHRONIZE CACHE (10) - VERIFY (10)

  4. UFS 增值可选命令: - READ (16), WRITE (16), PRE-FETCH (16), SYNCHRONIZE CACHE (16), READ CAPACITY (16) - 这些命令支持具有 8 字节 LBA 字段的更大容量逻辑单元

参考时钟 [Section 6.4, Table 6.4]_

M-PHY 规范定义参考时钟对于 State Machine Type I 为可选 [MIPI-M-PHY]。由于 PWM 信令是自时钟的，不需要参考时钟进行数据锁存。因此，UFS 设备应能在 LS-MODE (LINE-CFG、SLEEP 和 PWM-BURST) 下无参考时钟运行。

然而，参考时钟的存在可用于实现更低的 BER 和更快的 HS-MODE PLL/DLL锁定。因此，UFS 设备应实现方波单端参考时钟输入，并且在 HS-MODE (STALL 和 HS-BURST) 运行时需要本节所述特性的参考时钟。

为避免潜在的竞争条件，建议在请求功率模式变更到 Fast_Mode 或 FastAuto_Mode 之前，该参考时钟已经存在。

bRefClkFreq 属性:

  • 向设备指示 REF_CLK 信号的频率，默认值对应 26 MHz

  • UFS 设备只有在 bRefClkFreq 属性指示正确的 REF_CLK 频率值时才能在 HS-MODE 下运行

  • bRefClkFreq 属性只有在两个子链路都处于 LS-MODE 时才能写入

参考时钟门控:

参考时钟在以下情况下不需要且可以关闭:

  • 两个 SUB-LINK 都已达到并在以下 M-PHY 状态下运行: - LS-MODE (LINE-CFG、SLEEP 或 PWM-BURST 状态) - HIBERN8 状态

如果 UFS 主机发起从 HS-MODE 到 LS-MODE 或 HIBERN8 的功率模式变更，则应确保至少经过 bRefClkGatingWaitTime 定义的最短持续时间后才能关闭参考时钟 [Figure 6.3]_。

UFS 主机可在收到 HS-MODE 到 LS-MODE 转换的 DME_POWERMODE.ind 或 HS-MODE 到 HIBERN8 转换的 DME_HIBERNATE_ENTER.ind 时启动定时器。除了 bRefClkGatingWaitTime 外，还应考虑 Device PA_MinRxTrailingClocks 和 Host PA_MinRxTrailingClocks 来确定何时可以停止参考时钟。

在从 HS-MODE 到 HIBERN8 的转换以达到 UFS-DeepSleep 功率模式的情况下，也应应用此关闭参考时钟的约束。

UniPro 层定义使用 PA_INIT 机制的重新初始化过程。在此过程中，两个子链路都可能短暂进入 LS-MODE，然后返回 HS-MODE。在整个 PA_INIT 过程中不应门控参考时钟。

在从 LS-MODE (LINE-CFG 或 SLEEP) 或 HIBERN8 状态转换到 STALL 之前，参考时钟应开启并稳定运行。HS 模式下不应门控参考时钟。

复位、上电和掉电

复位类型 [Section 7.1]_

以下小节定义了复位 UFS 设备或其层的各种方式。

Power-on Reset [Section 7.1.1, Figure 7.1]_:

通过关闭再开启 VCCQ、VCCQ2 和 VCC 电源获得上电复位。

UFS 设备应有自己的上电检测电路，在上电后将 UFS 设备及其所有层置于定义的状态。

Hardware Reset [Section 7.1.2, Figure 7.2, 7.3]_:

为 UFS 设备定义了专用的硬件复位信号。

复位时序参数:

复位信号低电平有效。UFS 设备不应检测到 100 ns 或更短的正或负 RST_n 脉冲。UFS 设备应检测到大于或等于 1 µs 的正或负 RST_n 脉冲宽度。

EndPointReset [Section 7.1.3, Figure 7.4]_:

EndPointReset 功能在 MIPI UniPro 规范中定义。

从主机 Application Client 通过 DME_SAP 调用:

DME_ENDPOINTRESET.req = 1

设备管理器通过 DME_SAP 从设备 UniPro 接收 EndPointReset 功能调用并执行 EndPointReset 功能。

UFS 设备在收到 EndPointReset 时应完全复位自身: - UFS Flags (除上电复位 UFS Flags 外) - UFS Attributes (除上电复位 UFS Attributes 外) - UniPro attributes 复位到默认值 - 启动 UniPro 链路启动

由于 attributes 复位到默认值，设备可能需要重新配置。此外，从 UFS 设备下载 boot 代码是可选的，基于系统级条件。

UFS 主机应忽略从 UFS 设备收到的 EndPointReset。

Logical Unit Reset [Section 7.1.4, Figure 7.5]_:

Logical Unit Reset 功能在 SCSI Architectural Model [SAM] 中定义。此复位通过 10.9.8.4 节描述的 SCSI Task Management 功能触发。

LU 复位 (LU 0, ..., bMaxNumberLU 指定的最大 LU):

从主机 Application Client 通过 UTP_TM_SAP 调用:

Task management LOGICAL UNIT RESET (IN (I_T_L Nexus))

LU Task manager 应通过 UTP_TM_SAP 从设备 UTP 接收功能调用并执行 LU 复位功能。

注意: Logical Unit Reset 不将设备参数设置为默认值，因此不建议使用 Logical Unit Reset 为系统启动准备 UFS 设备。

Host UniPro Warm Reset [Section 7.1.5]_:

主机中的 UniPro Warm Reset 事件是 UFS 设备复位的间接原因。

主机系统复位自己的 UniPro 栈:主机的 UniPro 栈复位活动通过 DME_LINKLOST.ind 消息信号通知给 UFS 设备的 UniPro 栈。如果 UFS 设备从主机系统收到此 DME_LINKLOST.ind 消息，它应开始重新初始化自己的 UniPro 栈的过程。此外，所有 UFS 设备级活动应中止，所有逻辑单元中的任务队列列表应清除，UFS 功率模式应根据 bInitPowerMode 返回到 UFS-Sleep 功率模式或 Active 功率模式。
