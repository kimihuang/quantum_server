JESD220E UFS 3.1 UTP 传输协议层深度分析

本文档基于 JEDEC Universal Flash Storage (UFS) Version 3.1 规范 (JESD220E, January 2020) 第 10 章， 深入分析 UFS 传输协议 (UTP) 层的架构、事务处理、UPIU 格式和关键数据通路。

参考文档：docs/pdf/UFS/JESD220E_UFS3_1.md

UTP 层概述

UTP 架构定位 [Section 10.1]_

UFS 传输协议 (UTP) 层位于 UFS 架构的传输层，建立在 MIPI UniPro(数据链路层) 和 MIPI M-PHY(物理层) 之上。 UTP 采用 SCSI 架构模型 [SAM] 作为通用架构模型，使用 SAM 任务管理函数进行任务管理。

核心特性:

  • 客户端 - 服务器模型: Initiator(发起方，主机) 发送命令请求，Target(目标方，UFS 设备) 执行请求

  • 任务队列管理: 每个逻辑单元 (LU) 包含任务队列，支持并发处理多个任务

  • UPIU 事务: 设备间通信通过 UFS 协议信息单元 (UPIU) 消息实现

  • Task Tag 机制: 唯一标识每个任务，支持多任务并发追踪

UTP 在 UFS 架构中的位置

+----------------------------------+
|    UFS Application Layer (UAP)   |  <- SCSI 命令层、设备管理器
+----------------------------------+
               |
               v
+----------------------------------+
|  UFS Transport Protocol (UTP)    |  <- 本文档分析核心
|        (传输协议层)               |
+----------------------------------+
               |
               v
+----------------------------------+
|      UFS InterConnect (UIC)      |  <- UniPro 数据链路层
+----------------------------------+
               |
               v
+----------------------------------+
|         MIPI M-PHY               |  <- 物理层
+----------------------------------+

UTP 服务交付子系统 [Section 10.4]_

服务交付子系统是在 Initiator 和 Target 设备之间传输服务请求和响应的 I/O 系统。 UTP 定义为独立于服务交付子系统的协议，便于移植到不同的 I/O 系统。

当前实现:

  • 使用 MIPI UniPro 总线作为传输载体

  • 使用 MIPI M-PHY 作为物理层

  • UPIU 结构作为 UniPro 服务数据单元 (T_SDU) 传递

  • T_SDU 大小与 UPIU 大小完全相同

尺寸限制:

  • 最小 T_SDU: 32 字节 (基本头格式)

  • 最大 T_SDU: 65600 字节 (包含头、扩展头和数据段)

事务类型详解

NOP OUT/NOP IN: 连接探测 ping 操作，检查设备连接路径

COMMAND: Initiator 发送到 LU 的命令请求，包含 CDB(命令描述块)

RESPONSE: Target 返回的命令执行状态和响应信息

DATA OUT: Initiator 到 Target 的数据传输 (写操作)

DATA IN: Target 到 Initiator 的数据传输 (读操作)

TASK MANAGEMENT REQUEST: 携带 SAM 任务管理函数请求

TASK MANAGEMENT RESPONSE: 携带任务管理函数响应

READY TO TRANSFER: Target 准备好接收下一个 DATA OUT UPIU 时发送，包含 DMA 上下文

QUERY REQUEST: 读取/写入设备描述符、属性、标志等配置数据

QUERY RESPONSE: 响应查询请求，返回请求的描述符信息

REJECT: Target 无法解析或执行收到的 UPIU 时发送

基本头字段详解 [Section 10.6.2]_

Transaction Type (字节 0):

Bit 7: HD (Header CRC 使能，本版本固定为 0)
Bit 6: DD (Data CRC 使能，本版本固定为 0)
Bit 5-0: Transaction Code (事务代码)

Flags (字节 1): 根据事务类型变化

对于 COMMAND UPIU:

Bit 7: 保留
Bit 6: R (Read，数据从 Target 到 Initiator)
Bit 5: W (Write，数据从 Initiator 到 Target)
Bit 4: 保留
Bit 3: 保留
Bit 2: CP (Command Priority，命令优先级)
Bit 1-0: ATTR (Task Attribute，任务属性)

COMMAND UPIU 关键字段 [Section 10.7.1.1]_

Flags.R (Bit 6): - '1': 命令需要从 Target 到 Initiator 的数据传输 (读) - .R='1' 时.W 必须='0' - .R=.W='0' 时无需数据传输

Flags.W (Bit 5): - '1': 命令需要从 Initiator 到 Target 的数据传输 (写) - .W='1' 时.R 必须='0'

Expected Data Transfer Length (字节 12-15): - 表示完成 SCSI 命令所需传输的字节数 - 仅当 Flags.W 或 Flags.R 为'1'时有效 - 对于 SCSI READ/WRITE 命令，值为：逻辑块大小 × TRANSFER LENGTH

CDB (字节 16-31): 16 字节命令描述块 - 包含标准 SCSI CDB - CDB 大小由 CDB[0] 的操作码组位隐式指示 - 小于 16 字节的 CDB，未用字节保留

RESPONSE UPIU Flags [Section 10.7.2.1, Table 10.14]_

Flags.O (Overflow, Bit 6): - '1': 数据溢出，Target 有更多数据要传输 - Residual Transfer Count 表示未传输的字节数

DATA OUT 关键字段 [Section 10.7.3.1]_

Data Buffer Offset (字节 12-15): - 表示此 UPIU 数据载荷在完整传输区中的偏移 - Data Buffer Offset + Data Segment Length ≤ Expected Data Transfer Length - 支持乱序传输 (需设备支持并启用) - 对于 SCSI WRITE 命令，必须是逻辑块大小的整数倍

Data Transfer Count (字节 16-19): - 表示此 UPIU 中传输的字节数 - 最大值：65535 字节/UPIU - 必须与 Data Segment Length 字段值相同 - 与 Data Buffer Offset 一起构成 DMA 上下文

Data Segment (从字节 32 开始): - 包含实际数据载荷 - 始终 DWORD 对齐 - 最后一个 UPIU 可能需要填充到 DWORD 边界

DATA OUT 传输规则 [Section 10.7.13]_

基本规则:

  1. Initiator 在收到 READY TO TRANSFER UPIU 之前不得发送 DATA OUT UPIU

  2. Target 通过发送多个 READY TO TRANSFER UPIU 来 pacing 大数据传输

  3. 每个 DATA OUT UPIU 对应一个 READY TO TRANSFER UPIU

  4. Data Buffer Offset 和 Data Transfer Count 必须与 RTT 请求匹配

DATA IN 特性 [Section 10.7.4.1]_

与 DATA OUT 的区别:

  • 数据流向：Target → Initiator (读操作)

  • 无需 READY TO TRANSFER 握手

  • Host 预先分配足够的缓冲区接收全部数据

  • 支持乱序传输 (如果设备支持)

Data Buffer Offset: - 表示此 UPIU 数据在完整传输区中的偏移 - 允许非顺序传输 DATA IN UPIU - 对于 SCSI READ 命令，必须是逻辑块大小的整数倍

Data Transfer Count: - Target 放置在 Data Segment 中的有效字节数 - 最大值：65535 字节/UPIU - 必须与 Data Segment Length 相同

RTT UPIU 功能 [Section 10.7.5.1]_

用途: - Target 在处理需要 Data Out 传输的 SCSI 命令 (如写命令) 时发出 - 表示 Target 已准备好接收数据块 - 可顺序或乱序请求数据

关键约束:

  1. Data Buffer Offset: - 指示 Initiator 发送数据的起始位置 - 必须是 4 的整数倍 - 对于 SCSI WRITE 命令，必须是逻辑块大小的整数倍

  2. Data Transfer Count: - 表示 Target 请求的字节数 - 必须是 4 字节的整数倍 (最后一个 RTT 除外) - 不得超过 bMaxDataOutSize 属性值 - 对于 SCSI WRITE 命令，必须是逻辑块大小的整数倍

  3. 序列规则: - Target 可发送一个或多个 RTT UPIU 满足 Expected Data Transfer Length - Initiator 对每个 RTT 回复一个 DATA OUT UPIU - 支持乱序请求 (reverse order 等)

关键设计特性

数据 pacing 机制 [Section 10.2.2]_

问题: 设备可能有有限的缓冲资源或处理吞吐量

解决方案: READY TO TRANSFER UPIU

  • Target 在准备好接收下一个 DATA OUT UPIU 时发送 RTT

  • RTT 包含嵌入式传输上下文，用于在每个包基础上启动 DMA 传输

  • 允许 Target 控制数据流速率，避免缓冲区溢出

Data In 阶段: - 无需 RTT 握手 - 主机可预先指定 Data In 传输大小 - 提前分配适当的内存资源

Task Tag 关联机制 [Section 10.1, Table 10.9]_

同一任务的所有相关 UPIU 使用相同的 Task Tag:

Initiator UPIU

Target UPIU

NOP Out

NOP In

Command, Data Out

Ready to Transfer, Response

Command

Data In, Response

Task Management Request

Task Management Response

Query Request

Query Response

端到端 CRC 支持 (本版本未启用) [Section 10.6.2]_

HD Bit (Header CRC): - '1': 包含所有头段的端到端 CRC - 本版本不支持，HD 固定为'0'

DD Bit (Data CRC): - '1': 包含数据段的端到端 CRC - 本版本不支持，DD 固定为'0'

UPIU 尺寸灵活性 [Section 10.2.3, 10.3]_

最小尺寸: 32 字节 - 12 字节基本头 - 20 字节事务特定字段

最大尺寸: 65600 字节 - 包含基本头、可选扩展头、数据段 - Data Segment 最大 65535 字节

对齐要求: - 所有 UPIU 尺寸为 32 位 (4 字节) 的整数倍 - 数据段需要填充到最近的 DWORD 边界
