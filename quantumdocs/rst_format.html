1. reStructuredText 语法参考手册

reStructuredText (reST) 是 Sphinx 和 Docutils 使用的默认纯文本标记语言。Docutils 提供基本的 reStructuredText 语法，而 Sphinx 扩展了更多功能。

本文档涵盖 reStructuredText 的所有主要语法元素。

1.1. 段落与文本格式

1.1.1. 段落

段落是 reStructuredText 文档中最基本的块。段落是由一个或多个空行分隔的文本块。与 Python 一样，缩进在 reStructuredText 中很重要，因此同一段落的所有行必须左对齐到相同的缩进级别。

这是第一段。

这是第二段，与第一段之间有空行分隔。

1.1.2. 行内标记

标准 reStructuredText 行内标记非常简单：

  • 单星号：*文本* 表示 *斜体*（emphasis）

  • 双星号：**文本** 表示 **粗体**（strong）

  • 反引号：``文本`` 表示 ``代码``（literal）

注意事项：

  • 不能嵌套使用

  • 内容不能以空格开头或结尾，* 文本* 是错误的

  • 必须与非单词字符分隔。使用反斜杠转义空格来解决：这是\ *一个*\ 词

1.1.3. 转义字符

如果星号或反引号在文本中出现并可能与行内标记分隔符混淆，则必须用反斜杠转义：

使用 \* 转义星号，使用 \` 转义反引号。

1.2. 列表与引用块

1.2.1. 无序列表

列表标记很自然：只需在段落开头放置星号并正确缩进：

* 这是无序列表的第一项
* 这是第二项
  第二项使用两行
* 这是第三项

显示效果：

  • 这是无序列表的第一项

  • 这是第二项 第二项使用两行

  • 这是第三项

1.2.2. 有序列表

1. 这是有序列表的第一项
2. 这是第二项
#. 这也是有序列表（自动编号）
#. 自动编号的第二项

显示效果：

  1. 这是有序列表的第一项

  2. 这是第二项

1.2.3. 嵌套列表

嵌套列表是可能的，但要注意必须用空行与父列表项分隔：

* 这是
* 一个列表
  * 嵌套列表
  * 一些子项
* 父列表继续

显示效果：

  • 这是

  • 一个列表

    • 嵌套列表

    • 一些子项

  • 父列表继续

1.2.4. 定义列表

术语（最多一行文本）
   术语的定义，必须缩进
   甚至可以由多个段落组成

下一个术语
   描述。

显示效果：

术语（最多一行文本）

术语的定义，必须缩进 甚至可以由多个段落组成

下一个术语

描述。

用于命令行选项：

-a        选项 a 的说明

--long    长选项的说明

1.2.7. 引用段落

通过比周围段落缩进更多来创建引用段落：

    缩进比周围文本更多。

显示效果：

1.2.8. 行块

行块是保留换行符的一种方式：

| 源文件中完全一样。

显示效果：

1.3. 代码块与字面块

1.3.1. 字面块

字面代码块由段落末尾的特殊标记 :: 引入。字面块必须缩进：

这是普通文本段落。下一段是代码示例::

   它不会以任何方式处理，除了
   删除缩进。
   可以跨越多行。

这又是普通文本段落。

:: 标记的处理很智能：

  • 如果它作为一个独立的段落出现，该段落完全从文档中省略

  • 如果它前面有空格，标记被移除

  • 如果它前面是非空格字符，标记被替换为单个冒号

1.3.2. 代码块指令

使用 code-block 指令可以启用语法高亮：

.. code-block:: python
   :caption: Python 示例
   :linenos:

   def hello_world():
       print("Hello, World!")
       return True

显示效果：

Python 示例
1def hello_world():
2    print("Hello, World!")
3    return True

支持的语言包括：Python、C、C++、JavaScript、Java、Bash、HTML、CSS、SQL 等。

1.3.3. Doctest 块

Doctest 块是剪切粘贴到文档字符串中的交互式 Python 会话：

>>> 1 + 1
2
>>> print("Hello")
Hello

1.4.3. CSV 表格

使用逗号分隔值创建表格：

.. csv-table:: CSV 表格示例

     - 数据5
     - 数据6

1.5. 章节标题

章节标题由下划线（可选加上划线）标示，使用标点字符，长度至少与文本相同：

=================
这是一级标题
=================

-----------------
这是二级标题
-----------------

这是三级标题
=================

这是四级标题
-----------------

这是五级标题
~~~~~~~~~~~~~~~~~

这是六级标题
^^^^^^^^^^^^^^^^^

Python 开发者指南推荐的约定：

  • # 加上划线，用于部分（Parts）

  • * 加上划线，用于章节（Chapters）

  • = 用于节（Sections）

  • - 用于小节（Subsections）

  • ^ 用于小小节（Subsubsections）

  • " 用于段落（Paragraphs）

1.6. 超链接

1.6.1. 外部链接

文本中的 URL 和电子邮件地址会自动链接，无需特殊标记：

https://www.sphinx-doc.org/

显示效果：https://www.sphinx-doc.org/

1.6.2. 分隔式链接

将 URL 放在段落下方的最佳方式：

这是一个包含 `链接文本`_ 的段落。

.. _链接文本: https://www.sphinx-doc.org/

显示效果：

这是一个包含 链接文本 的段落。

1.6.3. 内联链接

将 URL 嵌入文本中：

`Sphinx 文档 <https://www.sphinx-doc.org/>`__

重要

链接文本与 URL 的左尖括号 < 之间必须有空格。

小技巧

嵌入 URL 时使用两个尾部下划线。技术上单下划线也可以，但会创建命名引用而不是匿名引用。

1.6.4. 内部链接

内部链接通过 Sphinx 提供的特殊角色实现：

.. _my-label:

这是一个标签位置。

稍后可以引用 :ref:`my-label`。

1.6.5. 交叉引用

Sphinx 提供多种交叉引用角色：

:doc:`install_rst`           # 引用文档
:ref:`标签名称`               # 引用标签
:meth:`类方法`                # 引用方法
:class:`类名`                 # 引用类
:func:`函数名`                # 引用函数
:mod:`模块名`                 # 引用模块
:option:`选项名`              # 引用命令行选项

1.7. 图片

1.7.1. 基本图片

.. image:: images/logo.png
   :alt: Logo 图片
   :width: 200px
   :height: 100px
   :align: center

图片选项：

  • alt: 替代文本

  • width: 宽度（像素或带单位如 200px 或 50%）

  • height: 高度

  • align: 对齐方式（top、middle、bottom、left、center、right）

  • scale: 缩放比例（百分比）

1.7.2. 图片通配符

Sphinx 扩展支持星号扩展名：

.. image:: gnu.*

Sphinx 会搜索匹配提供的模式的所有图像并确定其类型。每个构建器然后从这些候选中选择最佳图像。

1.7.3. Figure 指令

figure 指令可以添加标题：

.. figure:: images/example.png
   :alt: 示例图片
   :align: center
   :figwidth: 80%

   这是图片的标题说明。

1.8. 脚注与引用

1.8.1. 脚注

使用 [#name]_ 标记脚注位置，并在文档底部添加脚注内容：

这是一段文本 [#f1]_ 包含脚注引用 [#f2]_。

.. rubric:: 脚注

.. [#f1] 第一个脚注的内容。
.. [#f2] 第二个脚注的内容。

显示效果：

这是一段文本 [1] 包含脚注引用 [2]。

脚注

[1]

第一个脚注的内容。

[2]

第二个脚注的内容。

1.8.2. 自动编号脚注

这是一个自动编号脚注 [#]_。

.. [#] 自动编号的脚注内容。

1.8.3. 手动编号脚注

这是手动编号脚注 [1]_。

.. [1] 手动编号的脚注内容。

1.8.4. 引用

引用类似于脚注，但使用非数字标签：

这是引用文献 [Ref]_ 的示例。

.. [Ref] 书籍或文章引用，URL 或其他参考信息。

显示效果：

这是引用文献 [Ref] 的示例。

[Ref]

书籍或文章引用，URL 或其他参考信息。

1.9. 角色（Roles）

角色是行内显式标记的一种，表示包含的文本应以特定方式解释。一般语法是 :rolename:`content`。

1.9.1. 标准角色

Docutils 支持以下角色：

:emphasis: 等同于 *斜体*
:strong: 等同于 **粗体**
:literal: 等同于 ``代码``
:subscript: 下标文本
:superscript: 上标文本

显示效果：

emphasis:

等同于 斜体

strong:

等同于 粗体

literal:

等同于 代码

H2O 是水的化学式。

E = mc2是质能方程。

1.9.2. Sphinx 扩展角色

:doc:`文档名`           # 引用文档
:ref:`标签`             # 引用标签
:file:`文件名`          # 文件路径
:kbd:`Ctrl+C`           # 键盘按键
:code:`代码`            # 代码片段
:math:`E=mc^2`          # 数学公式
:guilabel:`标签`        # GUI 标签
:menuselection:`菜单项` # 菜单选择

1.10. 指令（Directives）

指令是显式标记的通用块。基本结构：

.. directive-name:: arguments
   :option1: value1
   :option2: value2

   内容段落...

1.10.1. 警告框

Docutils 支持以下警告框：

.. note:: 这是一个注释。

.. warning:: 这是一个警告。

.. tip:: 这是一个提示。

.. important:: 这是一个重要信息。

.. caution:: 这是一个注意事项。

.. danger:: 这是一个危险警告。

.. error:: 这是一个错误信息。

.. hint:: 这是一个提示信息。

显示效果：

备注

这是一个注释。

警告

这是一个警告。

小技巧

这是一个提示。

重要

这是一个重要信息。

1.10.2. 通用警告框

.. admonition:: 自定义标题
   :class: tip

   这是一个自定义警告框内容。

显示效果：

自定义标题

这是一个自定义警告框内容。

1.10.3. 内容指令

目录（局部）：

.. contents:: 目录
   :depth: 2
   :local:

容器：

.. container:: custom-class

   这是一个带自定义类的容器。

标题（无文档结构关系）：

.. rubric:: 这是一个小标题

解析字面块：

.. parsed-literal::

   这支持 *行内标记* 解析。

警句：

.. epigraph::

   这是一句警句。
   -- 作者名

摘要：

.. highlights::

   这是摘要内容。

引用块：

.. pull-quote::

   这是引用块内容。

复合段落：

.. compound::

   这是复合段落的第一部分。

   这是第二部分。

1.11. 替换

1.11.1. 替换定义

替换是通过 |name| 在文本中引用的文本和/或标记片段：

.. |version| replace:: 1.0.0
.. |date| date::

使用：

当前版本：|version|，发布日期：|date|。

1.11.2. 图片替换

.. |logo| image:: images/logo.png
          :alt: Logo

使用：

1.12. 注释

任何不是有效标记构造的显式标记块都被视为注释：

.. 这是一个单行注释。

..
   这是一个
   多行注释。
   注释不会出现在输出中。

1.13. HTML 元数据

meta 指令允许指定 Sphinx 文档页面的 HTML 元数据：

.. meta::
   :description: Sphinx 文档构建器
   :keywords: Sphinx, documentation, builder
   :author: 作者名

生成的 HTML：

<meta name="description" content="Sphinx 文档构建器">
<meta name="keywords" content="Sphinx, documentation, builder">

1.15. 常见问题

1.15.1. 行内标记分隔

行内标记必须与非单词字符分隔。使用反斜杠转义空格解决：

这是一个\ *强调*\ 词。

1.15.2. 不能嵌套

行内标记不能嵌套：

*参见 :func:`foo`*   # 这是无效的

正确做法：

*参见* :func:`foo`

1.15.3. 缩进一致性

所有相关内容必须使用一致的缩进。缩进不是固定数量的空格，而是相对于指令起始位置的任意数量的空白。
