    GDB 调试命令分级速查手册¶

    按 初级 → 中级 → 高级 → 生产级 四个层次组织，覆盖嵌入式 Linux 交叉调试全场景。

    一、初级 — 基本调试操作¶

    1.1 启动与退出¶

    命令 缩写 用途
    gdb ./app 调试本地程序
    gdb ./app core 加载 core dump
    gdb ./app <pid> attach 到进程
    gdb -q ./app 安静模式（不显示版本）
    gdb -tui ./app TUI 文本界面
    gdb -x init.gdb ./app 执行初始化脚本
    quit q 退出 GDB
    detach 脱离进程（不终止）

    1.2 远程调试连接¶

    命令 用途
    target remote localhost:1234 TCP 连接 gdbserver
    target remote /dev/ttyUSB0 串口连接
    target extended-remote localhost:1234 扩展远程模式（支持多进程）
    set sysroot /path/to/target 设置目标根文件系统
    set solib-search-path /path/to/libs 设置共享库搜索路径

    1.3 执行控制¶

    命令 缩写 用途
    run [args] r 启动程序
    run < input.txt 重定向输入
    set args arg1 arg2 设置运行参数
    continue [N] c 继续执行（跳过 N 次断点）
    step [N] s 源码级单步进入
    next [N] n 源码级单步跳过
    stepi [N] si 汇编级单步进入
    nexti [N] ni 汇编级单步跳过
    finish fin 执行到当前函数返回
    until [line] u 执行到指定行（跳出循环）
    advance <loc> 执行到指定位置
    set pagination off 关闭分页
    set confirm off 关闭确认提示

    1.4 断点¶

    命令 用途
    break main 函数断点
    break file.c:123 行号断点
    break *0x400800 地址断点
    tbreak main 临时断点（命中一次后删除）
    info breakpoints 查看所有断点
    delete N 删除断点 N
    delete 删除所有断点
    enable N 启用断点 N
    disable N 禁用断点 N
    clear main 清除 main 处的断点

    1.5 查看源码与变量¶

    命令 缩写 用途
    list l 查看源码
    list main 查看 main 函数源码
    list file.c:123 查看指定文件行
    print x p 打印变量
    print/x x 十六进制打印
    print *ptr 解引用指针
    print array[0]@10 打印数组前 10 个元素
    info locals 当前帧局部变量
    info args 当前帧函数参数

    1.6 调用栈¶

    命令 缩写 用途
    backtrace bt 显示调用栈
    backtrace full bt full 带局部变量的调用栈
    frame N f N 切换到第 N 帧
    up [N] 向上 N 帧（调用者方向）
    down [N] 向下 N 帧（被调用者方向）
    info frame 当前帧详细信息

    二、中级 — 进阶调试技巧¶

    2.1 条件断点与断点控制¶

    命令 用途
    break file.c:123 if x > 10 条件断点
    condition 1 x > 10 给已有断点加条件
    condition 1 清除断点条件
    ignore 1 100 忽略断点 1 共 100 次
    enable once 1 启用一次后自动禁用
    enable delete 1 启用一次后自动删除
    save breakpoints file.gdb 保存断点到文件
    source file.gdb 加载断点文件

    2.2 观察点（Watchpoint）¶

    命令 用途
    watch global_var 写观察点（变量变化时暂停）
    rwatch global_var 读观察点
    awatch global_var 读/写观察点
    watch *(int*)0x7fff1234 监视指定地址
    watch my_struct.field 监视结构体字段
    info watchpoints 查看观察点

    2.3 内存检查¶

    命令 用途
    x/16xw 0x400800 16 个 4 字节十六进制
    x/20i $pc 反汇编 20 条指令
    x/s (char*)ptr 打印字符串
    x/10hg &float_array 10 个 2 字节十六进制
    x/NFU addr 通用格式：N=数量 F=格式 U=单位

    x 命令格式说明：

    格式 (F) 说明 单位 (U) 说明
    x 十六进制 b 1 字节
    d 十进制 h 2 字节
    t 二进制 w 4 字节
    o 八进制 g 8 字节
    u 无符号
    f 浮点
    i 指令
    s 字符串
    a 地址

    2.4 寄存器与反汇编¶

    命令 用途
    info registers 所有寄存器
    info registers x0 x1 sp pc 指定寄存器
    print $pc 程序计数器
    print $sp 栈指针
    print $lr 链接寄存器 (ARM)
    set $x0 = 0 修改寄存器
    disassemble 当前函数反汇编
    disassemble main 指定函数反汇编
    disassemble /m main 混合源码和汇编
    disassemble /r main 显示原始字节
    disassemble 0x400800,0x400900 地址范围反汇编

    2.5 修改变量¶

    命令 用途
    set x = 10 修改变量值
    set *0x400800 = 0xFF 修改指定地址的值
    set {int}0x400800 = 42 按类型写入内存
    set my_struct.field = 3.14 修改结构体字段

    2.6 多线程调试¶

    命令 用途
    info threads 列出所有线程
    thread 3 切换到线程 3
    thread apply all bt 所有线程调用栈
    thread apply all info locals 所有线程局部变量
    break file.c:123 thread 3 仅线程 3 触发的断点
    set scheduler-locking on 仅当前线程执行
    set scheduler-locking step 单步时锁定，continue 解锁

    2.7 共享库与符号¶

    命令 用途
    info sharedlibrary 查看已加载共享库
    sharedlibrary 加载所有共享库符号
    sharedlibrary libmy 仅加载匹配的符号
    nosharedlibrary 卸载所有共享库符号
    set auto-solib-add on 自动加载共享库符号（默认）
    set sysroot /path/to/sysroot 设置目标根文件系统

    2.8 Core Dump 基础分析¶

    命令 用途
    gdb ./app core 加载 core dump
    core-file /path/to/core 加载 core 文件
    bt full 带变量的完整调用栈
    print $_siginfo 信号详细信息
    print $_siginfo.si_signo 信号编号
    print $_siginfo.si_code 信号代码
    print $_siginfo._sifields._sigfault.si_addr SIGSEGV 故障地址
    info threads core dump 中的线程
    thread apply all bt 所有线程调用栈

    2.9 gcore（运行时 Core Dump）¶

    命令 用途
    gcore /path/to/core 生成运行时 core dump
    detach 脱离进程（进程继续运行）
    gdbserver --attach :1234 <pid> 目标端 attach 到运行中进程

    三、高级 — 专业调试技术¶

    3.1 断点命令（自动化断点行为）¶

    # 断点命中时自动执行命令
commands 1
  printf "x = %d\n", x
  continue
end

# 非侵入式日志断点（不暂停程序）
commands 2
  silent
  printf "[LOG] entered func, x=%d\n", x
  continue
end

    3.2 条件断点优化¶

    # 慢速：每次命中都评估条件
break file.c:100 if i == 999999

# 快速：ignore 跳过前 N 次
break file.c:100
ignore 1 999998

# 计数器方式
set $count = 0
break file.c:100
commands
  silent
  set $count = $count + 1
  if $count == 999999
    printf "Hit after %d iterations\n", $count
  else
    continue
  end
end

    3.3 反向调试（Record/Replay）¶

    命令 用途
    record 开始完整记录
    record btrace 使用分支追踪（更高效）
    reverse-continue 反向继续
    reverse-step 反向单步进入
    reverse-next 反向单步跳过
    reverse-finish 反向执行到调用点
    record stop 暂停记录
    record delete 删除记录
    info record 查看记录状态

    3.4 多进程调试¶

    命令 用途
    set follow-fork-mode parent fork 后跟踪父进程（默认）
    set follow-fork-mode child fork 后跟踪子进程
    set detach-on-fork off 两个进程都在 GDB 控制下
    info inferiors 查看被 GDB 控制的进程
    inferior 2 切换到 inferior 2
    catch fork fork 时暂停
    catch vfork vfork 时暂停
    catch exec exec 时暂停
    catch syscall open open 系统调用时暂停

    3.5 信号处理¶

    命令 用途
    info signals 查看所有信号处理设置
    handle SIGSEGV nostop SIGSEGV 时不暂停
    handle SIGSEGV stop SIGSEGV 时暂停
    handle SIGUSR1 pass SIGUSR1 传递给程序
    handle SIGUSR1 nopass SIGUSR1 不传递给程序
    catch signal SIGUSR1 捕获指定信号
    signal SIGUSR1 手动发送信号给程序

    3.6 便捷变量与历史值¶

    命令 用途
    set $my_var = x + 1 设置便捷变量
    print $my_var 使用便捷变量
    print $ 上一次 print 的值
    print $$ 上上次 print 的值
    set $_ = $pc GDB 内部便捷变量

    3.7 非侵入式调试¶

    # 调用程序函数（有副作用！）
call printf("value = %d\n", my_var)
call fflush(stdout)
call malloc_stats()

# 更安全：直接 print 表达式
print my_var
print my_struct.field

    3.8 内核模块调试¶

    命令 用途
    add-symbol-file my_module.ko 0xffffffc00a00000 加载模块符号
    add-symbol-file my_module.ko 0xa00000 -s .data 0xa01000 -s .bss 0xa02000 指定段地址加载
    info files 查看加载的段信息

    3.9 C++ 调试¶

    命令 用途
    catch throw 捕获异常抛出
    catch throw MyException 捕获指定类型异常
    catch catch 捕获异常捕获
    print *obj 查看对象（需 print object on）
    print /a *(void**)obj 查看虚表指针
    x/10a *(void**)obj 查看虚表内容

    3.10 STL 容器查看¶

    命令 用途
    print my_vector std::vector（需 pretty-printer）
    print *my_vector._M_impl._M_start@10 vector 前 10 个元素
    print my_string.c_str() std::string C 字符串
    print my_ptr.use_count() shared_ptr 引用计数
    print *my_ptr 解引用智能指针

    4.3 GDB Python 脚本¶

    # 自动日志断点（不暂停，仅记录）
python
import gdb

class LogBreakpoint(gdb.Breakpoint):
    def __init__(self, spec, expr):
        super().__init__(spec)
        self.expr = expr

    def stop(self):
        val = gdb.parse_and_eval(self.expr)
        print("[LOG] %s = %s" % (self.expr, val))
        return False  # 不暂停

LogBreakpoint("main.c:42", "x")
LogBreakpoint("main.c:100", "data->len")
end

# 自定义 pretty-printer
python
class MyStructPrinter:
    def __init__(self, val):
        self.val = val

    def to_string(self):
        return "my_dev(status=%d, count=%d)" % (
            int(self.val['status']),
            int(self.val['count']))

def my_struct_lookup(val):
    if str(val.type) == 'struct my_dev':
        return MyStructPrinter(val)
    return None

gdb.pretty_printers.append(my_struct_lookup)
end

# 批量分析 core dump
python
import gdb
import json

def analyze_core():
    result = {}
    result['backtrace'] = gdb.execute("bt", to_string=True)
    result['registers'] = gdb.execute("info registers", to_string=True)
    result['threads'] = gdb.execute("thread apply all bt", to_string=True)
    try:
        result['siginfo'] = gdb.execute("print $_siginfo", to_string=True)
    except:
        result['siginfo'] = "N/A (gcore dump)"
    with open('/tmp/core_analysis.json', 'w') as f:
        json.dump(result, f, indent=2)
    print("Analysis saved to /tmp/core_analysis.json")

analyze_core()
end

    4.4 生产环境 Core Dump 分析流水线¶

    #!/bin/bash
# core_analyze.sh - 自动化 core dump 分析脚本

GDB=/path/to/aarch64-none-linux-gnu-gdb
BINARY=$1
CORE=$2
SYSROOT=/path/to/target
REPORT="/tmp/core_report_$(date +%Y%m%d_%H%M%S).txt"

if [ -z "$BINARY" ] || [ -z "$CORE" ]; then
    echo "Usage: $0 <binary> <core_file>"
    exit 1
fi

echo "=== Core Dump Analysis Report ===" > "$REPORT"
echo "Binary: $BINARY" >> "$REPORT"
echo "Core:   $CORE" >> "$REPORT"
echo "Date:   $(date)" >> "$REPORT"
echo "" >> "$REPORT"

$GDB -batch \
  -ex "set pagination off" \
  -ex "set print pretty on" \
  -ex "set sysroot $SYSROOT" \
  -ex "file $BINARY" \
  -ex "core-file $CORE" \
  -ex "printf '\n=== Signal Info ===\n'" \
  -ex "print \$_siginfo.si_signo" \
  -ex "print \$_siginfo.si_code" \
  -ex "print \$_siginfo._sifields._sigfault.si_addr" \
  -ex "printf '\n=== Backtrace ===\n'" \
  -ex "bt full" \
  -ex "printf '\n=== All Threads ===\n'" \
  -ex "thread apply all bt" \
  -ex "printf '\n=== Registers ===\n'" \
  -ex "info registers" \
  -ex "printf '\n=== Disassembly ===\n'" \
  -ex "frame 0" \
  -ex "disassemble /m" \
  -ex "quit" \
  >> "$REPORT" 2>&1

echo "Report saved to $REPORT"

    4.5 Core Dump 过滤与精简¶

    # 设置 core dump 过滤器（控制哪些内存区域写入 core）
set coredump-filter 0x3f    # 所有区域（默认）
set coredump-filter 0x01    # 仅匿名私有内存（最小）
set coredump-filter 0x0f    # 匿名 + 文件映射私有

# 过滤器位说明
# bit 0 (0x01): 匿名私有内存
# bit 1 (0x02): 匿名共享内存
# bit 2 (0x04): 文件映射私有内存
# bit 3 (0x08): 文件映射共享内存
# bit 4 (0x10): ELF 头
# bit 5 (0x20): 大页私有内存

    4.7 gcore 定时抓取脚本¶

    #!/bin/bash
# periodic_gcore.sh - 定时 gcore 抓取
# 用法: ./periodic_gcore.sh <pid> <interval_sec> <count>

PID=$1
INTERVAL=${2:-60}
COUNT=${3:-5}
GDB=/path/to/aarch64-none-linux-gnu-gdb
BINARY=/path/to/app
SYSROOT=/path/to/target

for i in $(seq 1 $COUNT); do
    TIMESTAMP=$(date +%Y%m%d_%H%M%S)
    OUTPUT="/tmp/core_${PID}_${TIMESTAMP}"

    # 启动临时 gdbserver
    gdbserver --attach :12345 $PID &
    GDBSERV_PID=$!
    sleep 1

    # gcore
    $GDB -batch \
      -ex "set sysroot $SYSROOT" \
      -ex "target remote localhost:12345" \
      -ex "gcore $OUTPUT" \
      -ex "detach" \
      -ex "quit" \
      $BINARY 2>/dev/null

    kill $GDBSERV_PID 2>/dev/null
    wait $GDBSERV_PID 2>/dev/null

    echo "[$i/$COUNT] Saved: $OUTPUT ($(stat -c%s $OUTPUT) bytes)"

    if [ $i -lt $COUNT ]; then
        sleep $INTERVAL
    fi
done

    4.8 常用生产级命令速查¶

    场景 命令序列
    分析 core dump gdb app core → bt full → print $_siginfo → thread apply all bt
    找死锁 thread apply all bt → 搜索 __lll_lock_wait/pthread_mutex_lock
    找内存越界 watch *ptr 或 break memcpy + 条件检查
    找空指针 print $_siginfo._sifields._sigfault.si_addr → frame N 回溯
    找竞态 watch shared_var + set scheduler-locking step
    调试子进程 set follow-fork-mode child 或 catch fork
    运行时抓取 gdbserver --attach :1234 <pid> → gcore → detach
    批处理分析 gdb -batch -ex "bt full" -ex "thread apply all bt" app core
    自动化断点 commands N + silent + printf + continue
    多次对比 gcore → 等待 → gcore → 对比全局变量/堆状态

    五、信号与故障速查¶

    5.1 常见信号¶

    信号 编号 含义 典型原因
    SIGSEGV 11 段错误 空指针、野指针、越界、栈溢出
    SIGABRT 6 异常终止 abort()、assert、double-free、堆损坏
    SIGBUS 7 总线错误 未对齐访问、mmap 错误
    SIGFPE 8 浮点异常 整数除零、浮点溢出
    SIGILL 4 非法指令 函数指针损坏、代码段被覆写
    SIGPIPE 13 管道断裂 写入已关闭的 socket/pipe

    5.2 SIGSEGV 的 si_code¶

    si_code 宏名 含义
    1 SEGV_MAPERR 地址未映射（空指针/非法地址）
    2 SEGV_ACCERR 权限不足（写只读内存）

    5.3 退出码与信号¶

    退出码 信号
    139 (128+11) SIGSEGV
    134 (128+6) SIGABRT
    136 (128+8) SIGFPE
    135 (128+7) SIGBUS
    132 (128+4) SIGILL

    六、ARM64 寄存器速查¶

    寄存器 用途 说明
    x0-x7 参数/返回值 函数调用参数和返回值
    x8 间接结果 返回大结构体时指向内存
    x9-x15 临时寄存器 调用者保存
    x16-x17 IP0/IP1 链接器/PLT 临时使用
    x18 平台寄存器 平台特定用途
    x19-x28 被调用者保存 函数需保存恢复
    x29 (FP) 帧指针 指向当前栈帧
    x30 (LR) 链接寄存器 函数返回地址
    sp 栈指针 当前栈顶
    pc 程序计数器 当前执行地址
    cpsr 状态寄存器 条件标志 + EL 等
