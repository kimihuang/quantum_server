    1 概述¶

    1.1 技术背景¶

    内存安全漏洞（如缓冲区溢出、释放后使用（Use-After-Free）、越界访问）是系统安全的主要威胁之一。传统的内存安全机制（如 ASLR、Stack Canary、NX bit）提供了部分缓解措施，但无法从根本上检测运行时的内存越界访问。

    ARM 内存标记扩展（Memory Tagging Extension, MTE）是一种硬件辅助的内存安全机制，通过在内存中存储标记（Tag）并在指针中携带标记，在运行时自动检查内存访问的合法性。

    1.2 设计目标¶

    目标 描述
    越界访问检测 检测缓冲区溢出（堆/栈/全局）和越界读写
    释放后使用检测 检测对已释放内存的非法访问
    低性能开销 硬件自动检查，减少软件干预
    渐进式部署 支持同步故障、异步故障和仅记录模式
    向后兼容 未标记代码在标记内存上正常运行

    1.3 特性演进¶

    FEAT_MTE      → 基础 MTE（标记检查操作、分配标记读取）
FEAT_MTE2     → 增强版（支持分配标记的显式加载/存储指令）
FEAT_MTE_ASYNC → 异步故障模式（TFSR 寄存器记录故障位）
FEAT_MTE3     → 非对称故障（写异步 + 读同步）
FEAT_MTE_STORE_ONLY → 仅存储标记检查
FEAT_MTE_CANONICAL_TAGS → 规范标记（基于地址高位自动生成标记）

    2 架构设计¶

    2.1 核心概念¶

    MTE 基于以下四个核心概念构建：

    2.1.1 标记粒度（Tag Granule）¶

    内存中每 16 字节自然对齐的块称为一个标记粒度。这是 MTE 的基本操作单元。

    地址空间（每行 16 字节，一个 Tag Granule）
┌──────────────┐ ┌──────────────┐ ┌──────────────┐
│  Byte 0-15   │ │  Byte 16-31  │ │  Byte 32-47  │  ...
└──────────────┘ └──────────────┘ └──────────────┘
   Tag Granule 0   Tag Granule 1   Tag Granule 2
       ↓                ↓                ↓
   Alloc Tag 0      Alloc Tag 1      Alloc Tag 2

    2.1.2 分配标记（Allocation Tag）¶

    每个标记粒度在内存中关联一个 4 位分配标记（值为 0x0 ~ 0xF）。分配标记由内存分配器在分配内存时设置，并在释放时清除。

    关键属性： - 分配标记是一个 4-bit 值（16 种可能值） - 对分配标记的所有访问都是单拷贝原子（single-copy atomic）的 - 对不同分配标记的访问之间不保证原子性 - 指令在 VA 处显式访问分配标记时，实际访问的是按标记粒度对齐的地址

    2.1.3 逻辑地址标记（Logical Address Tag）¶

    64 位虚拟地址（VA）的 bits[59:56] 构成逻辑地址标记。当程序通过带标记的指针访问内存时，处理器提取指针中的逻辑地址标记，与目标内存的分配标记进行比较。

    64-bit 虚拟地址结构：
┌─────┬──────────┬───────────┬──────┬─────┐
│     │          │           │      │     │
│ 63  │ 62:60    │ 59:56     │ 55:0 │     │
│     │ Top Byte │ Tag [4b]  │ VA   │     │
└─────┴──────────┴───────────┴──────┴─────┘
              ↑
        逻辑地址标记
      （Logical Address Tag）

    2.1.4 标记检查操作（Tag Check Operation）¶

    标记检查操作是 MTE 的核心机制：

    1. 隐式读取目标地址对齐到标记粒度处的分配标记
    2. 比较分配标记与逻辑地址标记
    3. 不匹配则产生标记检查故障（Tag Check Fault）

    2.2 内存区域标记类型¶

    每个内存区域具有以下三种标记类型之一：

    类型 条件 标记检查行为
    已标记（Tagged） 实现了 FEAT_MTE2 + 启用了分配标记访问 + 阶段 1 属性定义了 Tagged 属性 + 内存属性为普通内存/内部外部写回/非瞬态/读写分配 执行标记检查操作
    规范标记（Canonically Tagged） 实现了 FEAT_MTE_CANONICAL_TAGS + 启用了规范标记 + 阶段 1 属性未定义 Tagged 属性 执行规范标记检查操作
    无标记（Untagged） 以上条件均不满足，或内存类型为设备内存 不执行标记检查

    已标记的完整条件¶

    内存区域为已标记需要同时满足以下条件：

    1. 实现了 FEAT_MTE2
    2. 该内存区域启用了分配标记访问（通过 SCTLR_ELx.ATA 等控制）
    3. 阶段 1 属性定义了 Tagged 属性
    4. 阶段 1 和阶段 2 转换的联合效果将内存属性定义为：普通内存、内部写回、外部写回、非瞬态、读分配、写分配

    特殊情况¶

    • 不可共享（Non-shareable）属性：内存区域被处理为已标记还是无标记由 IMPLEMENTATION DEFINED 决定
    • SCTLR_ELx.C = 0（缓存禁用）：内存区域被处理为已标记还是无标记由 CONSTRAINED UNPREDICTABLE 决定
    • Arm 建议：当 SCTLR_ELx.C = 0 时，软件应将 SCTLR_ELx.ATA 设为 0 以确保行为一致

    2.3 标记检查机制¶

    2.3.1 已标记区域的标记检查¶

    对已标记内存区域的标记检查操作：

    1. 隐式读取目标 VA 按标记粒度对齐处的分配标记
    2. 比较分配标记与 VA bits[59:56]（逻辑地址标记）
    3. 不匹配 → 产生标记检查故障

    2.3.2 规范标记区域的规范标记检查¶

    规范标记检查不比较分配标记，而是验证逻辑地址标记是否符合规范值：

    VA 范围配置 VA bit[55] 期望的逻辑地址标记
    两个 VA 范围 0 0b0000
    两个 VA 范围 1 0b1111
    单个 VA 范围 — 0b0000

    这确保了用户空间（低地址）和内核空间（高地址）的指针不会被混淆。

    2.3.3 无标记区域¶

    不执行任何标记检查。分配标记的读取返回 0b0000，分配标记的写入被忽略。

    2.4 标记检查内存访问的判定规则¶

    单拷贝原子显式内存访问在以下情况下为标记未检查（否则为标记检查）：

    2.4.1 访问类型排除¶

    以下访问类型始终为标记未检查：

    访问类型 说明
    指令取指 —
    对无标记内存区域的访问 —
    显式加载/存储分配标记的指令 如 LDG、STG、STGM、STZGM
    标记检查操作隐式引起的分配标记读取 —
    除 DC ZVA 之外的按 VA 的缓存维护操作 —
    页表遍历 —
    预取指令 —
    相对于 VNCR_EL2 的加载/存储 如果实现了 FEAT_NV2
    跟踪缓冲单元产生的访问 如果实现了 FEAT_TRBE
    统计剖析单元产生的访问 如果实现了 SPE
    GPT 信息取指 如果实现了 FEAT_RME
    受控保护栈数据访问 如果实现了 FEAT_GCS
    读取操作 如果在 ELx 启用了 FEAT_MTE_STORE_ONLY

    2.4.2 寻址模式排除¶

    使用以下寻址模式的指令产生的内存访问为标记未检查：

    • 仅基址寄存器，其中 SP 为基址寄存器
    • 基址寄存器加立即数偏移，其中 SP 为基址寄存器
    • 字面量（PC-relative）

    2.4.3 运行时控制排除¶

    • PSTATE.TCO = 1（临时绕过标记检查）
    • 逻辑地址标记被禁用（通过 TCR_ELx.TBI/MTX 控制）
    • TCR_ELx.TCMA = 1 且逻辑地址标记为 0b0000（免除全零标记的检查）
    • TCR_ELx.TCMA1 = 1 且访问高地址范围且逻辑地址标记为 0b1111（免除内核空间全一标记的检查）

    2.5 分配标记访问控制¶

    分配标记访问控制用于：

    1. 确定对内存中分配标记的访问权限
    2. 启用内存访问的标记检查
    3. 控制指令将逻辑地址标记插入地址的行为

    表：各异常级别的分配标记访问控制

    异常级别 系统寄存器控制
    EL3 SCTLR_EL3.ATA
    EL2 SCR_EL3.ATA, SCTLR_EL2.ATA
    EL1 SCR_EL3.ATA, HCR_EL2.ATA, SCTLR_EL1.ATA
    EL0 SCR_EL3.ATA, HCR_EL2.ATA, SCTLR_ELx.ATA0

    层级控制模型：上层异常级别可以通过控制位向下传播启用或禁用分配标记访问。例如 SCR_EL3.ATA 可以控制 EL2/EL1/EL0 是否允许分配标记访问。

    禁用时的行为：当分配标记访问被禁用时，将逻辑地址标记插入地址的指令会将分配标记视为零。

    3 硬件设计与实现¶

    3.1 标记粒度与标记存储¶

    3.1.1 标记粒度¶

    • 大小：16 字节（自然对齐）
    • 标记宽度：4 bit（0x0 ~ 0xF）
    • 标记密度：4 bit / 16 Byte = 1 bit / 4 Byte = 25% 地址空间开销（理论上）

    3.1.2 分配标记存储¶

    为常规内存（Conventional memory）的每个标记粒度提供分配标记存储空间。

    实现选项：

    方案 描述 优点 缺点
    Tag RAM 与 Data RAM 并行 专用标记存储与数据存储分离 独立控制，支持加密 额外存储面积
    数据存储内嵌标记 标记嵌入数据缓存行中 无额外存储 减少可用数据带宽
    数据地址空间映射 标记存储在独立的 PA 地址 软件可访问标记 需要额外的地址映射

    未提供标记存储时：对分配标记的访问结果由 IMPLEMENTATION DEFINED 决定。

    3.1.3 数据位置访问分配标记存储¶

    如果实现支持在独立的 PA 处通过数据内存效果访问分配标记存储：

    • 在标记位置对分配标记存储的访问与在数据位置的访问不需要在架构上保持一致（coherent）
    • 可以通过缓存维护操作使两个位置的访问互相可见
    • 如果实现了 FEAT_RME，仅在根 PA 空间可访问
    • 如果实现了 FEAT_MEC，使用默认 MECID 值零

    3.2 物理标记位置¶

    3.2.1 地址空间关系¶

    • 标记位置（Tag location）是 PA 空间中与某个地址关联的分配标记
    • 分配标记和同一 PA 处的字节是独立的位置
    • 没有机制可以在标记粒度起始 PA 之外的 PA 处访问分配标记

    3.2.2 转换特性¶

    对分配标记的隐式或显式访问： - 使用与对关联标记粒度的访问相同的地址转换 - 具有相同的转换效果

    3.2.3 与安全扩展的交互¶

    扩展 行为
    FEAT_RME 对标记位置的访问需要经过粒度保护检查
    FEAT_MEC 对标记位置的访问使用与关联标记粒度相同的 PA 空间和 MECID
    FEAT_MEC 主存中分配标记的加密是可选的纵深防御能力

    3.3 分配标记的缓存¶

    3.3.1 驱逐行为¶

    当数据从缓存中驱逐时，被驱逐的数据可能覆盖内存中被其他观察者写入的数据，条件是：

    • 数据已被该内存位置的可共享性域中的观察者写入
    • 或关联的分配标记已被该域中的观察者写入

    可被覆盖的最大内存大小由 CTR_EL0.DWG（Cache Write-Back Granule）定义。

    3.3.2 缓存维护要求¶

    如果实现可以覆盖内存中被其他观察者（非同可共享性域）写入的分配标记，则：

    缓存维护操作 对分配标记的要求
    清洗（Clean） 必须同时清洗关联的分配标记
    使失效（Invalidate） 必须同时清洗并使关联的分配标记失效
    清洗并使失效（Clean & Invalidate） 必须同时清洗并使关联的分配标记失效

    3.3.3 无标记区域的缓存维护¶

    如果内存区域为无标记，会使该区域中分配标记失效的数据缓存失效操作会清洗并使分配标记失效（而非仅使失效）。

    3.4 标记检查故障处理¶

    通过 SCTLR_ELx.{TCF, TCF0} 配置标记检查故障的行为：

    TCF/TCF0 值 行为 适用特性
    0b00 无影响（忽略故障） 所有
    0b01 产生同步标记检查故障异常 所有
    0b10 异步设置 TFSR_ELx/TFSRE0_EL1 中的位 需要 FEAT_MTE_ASYNC
    0b11 写操作异步故障；读操作同步故障 需要 FEAT_MTE3

    关键行为¶

    • 异步故障不影响数据访问：加载或存储正常完成，副作用照常发生
    • 规范标记检查故障与标记检查故障以相同方式处理
    • TCF 控制内核空间（TTBR1），TCF0 控制用户空间（TTBR0）

    异常综合征寄存器¶

    标记检查故障的异常通过标准 Data Abort 异常报告：

    字段 说明
    ESR_ELx.EC 异常类，指示标记检查故障类型
    ESR_ELx.ISV 指令特定故障
    ESR_ELx.TnD 如果为 1，表示是标记访问故障（非数据访问故障）
    FAR_ELx 故障地址

    3.5 异步标记检查故障¶

    需要实现 FEAT_MTE_ASYNC。

    3.5.1 标记故障状态寄存器¶

    寄存器 适用范围
    TFSR_EL1 EL1 及以上
    TFSR_EL2 EL2
    TFSR_EL3 EL3
    TFSRE0_EL1 EL0

    每个寄存器包含多个状态位（TF0 ~ TF15），对应 16 种可能的标记值。

    3.5.2 同步机制¶

    异步标记检查故障引起的对标记故障状态寄存器的隐式写入可以通过以下方式同步：

    1. 异常入口：如果 SCTLR_ELx.ITFSB = 1，进入 ELx 的异常入口时同步
    2. DSB 指令：在程序顺序中，在引起故障的指令之后执行非共享域上的 DSB

    3.5.3 与同步数据中止的交互¶

    如果一条指令同时产生同步数据中止和异步标记检查故障，且写入标记故障状态寄存器的位的值为 0，则该位变为 UNKNOWN。

    3.5.4 寄存器写入的竞态¶

    如果异步标记检查故障尚未被观测到且未被同步，在此时修改影响标记检查故障处理的系统寄存器，故障效果在旧值和新值之间由 CONSTRAINED UNPREDICTABLE 决定。

    3.5.5 SVE 相关约束¶

    对于 SVE Non-fault 和 SVE First-fault 加载指令：

    • 仅第一个活动元素（First active element）的异步标记检查故障会在标记故障状态寄存器中设置位
    • 非第一个活动元素的故障不设置位

    3.6 与设备内存的交互¶

    • 设备内存类型的内存区域始终为无标记
    • 对设备内存执行 STZGM 是 CONSTRAINED UNPREDICTABLE：可能清零数据并保持标记不变，也可能产生对齐故障
    • 对设备内存执行 DC GVA 是 CONSTRAINED UNPREDICTABLE：可能不修改标记，也可能产生对齐故障

    4.2 MTE 相关指令¶

    4.2.1 分配标记操作指令¶

    指令 功能 说明
    LDG Xn, [Xn, #imm] 加载标记 从内存读取分配标记，插入到地址 Xn 的高 4 位
    STG Xn, [Xn, #imm] 存储标记 将地址 Xn 的高 4 位（逻辑地址标记）写入内存的分配标记
    STGM Xn, [Xm, #imm] 存储标记（多寄存器） 批量存储分配标记
    STZGM Xn, [Xn, #imm] 存储标记并清零 将分配标记写入内存，同时将对应的 16 字节数据清零
    ST2G, STZ2G 双标记存储 同时操作两个标记粒度的分配标记
    ST4G, STZ4G 四标记存储 同时操作四个标记粒度的分配标记
    LDGM Xm, [Xn] 加载多个标记 从内存批量加载分配标记到通用寄存器对
    STGM Xm, [Xn] 存储多个标记 将通用寄存器对中的分配标记批量存储到内存

    4.2.2 地址生成指令¶

    指令 功能 说明
    ADDG Xd, Xn, #uimm6, #tag_offset 带标记偏移的加法 调整地址和标记，常用于数组索引
    SUBG Xd, Xn, #uimm6, #tag_offset 带标记偏移的减法 调整地址和标记

    4.2.3 缓存维护指令¶

    指令 功能 说明
    DC GVA, Xt 清除标记的虚拟地址 清除地址 Xt 处的分配标记缓存（不清洗到内存）
    DC GZVA, Xt 清除标记并清零的虚拟地址 清除标记并将数据清零
    DC IGVA, Xt 使标记的虚拟地址失效 使地址 Xt 处的分配标记缓存失效
    DC CIGVA, Xt 清洗并使标记的虚拟地址失效 清洗到内存并使分配标记缓存失效

    4.3 内核态使能 MTE¶

    4.3.1 启用步骤¶

    1. EL3（Secure Monitor）
   - SCR_EL3.ATA = 1    // 允许低特权级使用分配标记访问

2. EL2（Hypervisor）
   - HCR_EL2.ATA = 1    // 允许 EL1/EL0 使用分配标记访问
   - HCR_EL2.{TCF, TCF0} = 0b10  // 异步故障模式（推荐用于生产环境）
   - 或 HCR_EL2.{TCF, TCF0} = 0b01  // 同步故障模式（推荐用于调试）
   - TCR_EL2.MTE2 = 1   // 启用内核页表中的标记属性检查

3. EL1（内核）
   - SCTLR_EL1.ATA = 1  // 启用 EL1 分配标记访问
   - SCTLR_EL1.ATA0 = 1 // 启用 EL0 分配标记访问
   - SCTLR_EL1.{TCF, TCF0} = 0b10  // 异步故障模式
   - TCR_EL1.TBI0 = 1   // 启用用户空间逻辑地址标记
   - TCR_EL1.TBI1 = 1   // 启用内核空间逻辑地址标记
   - TCR_EL1.MTE0 = 1   // 启用用户空间页表标记属性
   - TCR_EL1.MTE1 = 1   // 启用内核空间页表标记属性
   - TCR_EL1.TCMA0 = 0  // 用户空间不禁用零标记检查
   - TCR_EL1.TCMA1 = 1  // 内核空间免除全一标记检查（可选）

    4.3.2 页表配置¶

    MTE 页表属性通过 MAIR_ELx 和页表描述符配置：

    MAIR_ELx.Attr[n] 设置：
  - Attr[3:0] 设为 0xF (Normal memory, Write-Back, RA/WA, Non-Transient)
  - Attr[5:4] 设为 0b00 (Inner WB, RA/WA)
  - Attr[7:6] 设为 0b00 (Outer WB, RA/WA)

页表描述符 Level 3 设置：
  - Block/Page descriptor bits [5:2] = MAIR 索引
  - Block/Page descriptor bit [53] = 1 (Tagged 属性)

    4.4 用户态使用 MTE¶

    4.4.1 编译器支持¶

    GCC 和 Clang 提供 MTE 相关编译选项：

    # GCC/Clang 编译选项
-fmemory-model=tagged       // 启用标记感知的内存模型
-march=armv8.5-a+memtag     // 启用 MTE 指令集
-march=armv8.5-a+memtag-tag // 仅使用标记操作指令（不插入检查）

    4.4.2 内存分配器集成¶

    // 堆分配器伪代码
void *malloc_tagged(size_t size) {
    // 1. 分配对齐到 16 字节的内存
    void *ptr = aligned_alloc(16, size);
    size_t tagged_size = ALIGN_UP(size, 16);

    // 2. 生成随机标记
    uint8_t tag = random_tag();

    // 3. 使用 STZGM 将标记写入所有标记粒度，同时清零数据
    for (size_t i = 0; i < tagged_size; i += 16) {
        stg_set_tag(ptr + i, tag);
        memset(ptr + i, 0, 16);  // 或使用 STZGM
    }

    // 4. 将标记插入返回的指针
    return insert_tag(ptr, tag);
}

// 释放时清除标记
void free_tagged(void *ptr) {
    uint8_t tag = get_tag(ptr);
    void *untagged_ptr = remove_tag(ptr);
    size_t size = get_alloc_size(untagged_ptr);
    size_t tagged_size = ALIGN_UP(size, 16);

    // 清除所有分配标记为 0
    for (size_t i = 0; i < tagged_size; i += 16) {
        stg_set_tag(untagged_ptr + i, 0);
    }

    free(untagged_ptr);
}

    4.4.3 异步故障的用户态处理¶

    // Linux 用户态异步 MTE 故障处理
// 通过 prctl 或信号处理

// 启用 MTE 异步故障
prctl(PR_SET_TAGGED_ADDR_CTRL, PR_TAGGED_ADDR_ENABLE, 0, 0, 0);

// 读取故障状态（通过 prctl）
unsigned long tags;
prctl(PR_MTE_GET_TAGS, &tags, 0, 0, 0);
// tags 中设置了对应位表示发生了该标记值的故障

// 清除故障状态
prctl(PR_MTE_SET_TAGS, 0, 0, 0, 0);

    4.5 规范标记模式¶

    规范标记（FEAT_MTE_CANONICAL_TAGS）是一种轻量级模式，不需要完整的分配标记支持：

    • 用户空间（低地址，bit[55]=0）：逻辑地址标记必须为 0b0000
    • 内核空间（高地址，bit[55]=1）：逻辑地址标记必须为 0b1111

    配置方式： 

    // 两个 VA 范围
SCTLR_ELx.{ATA, ATA0}  // 启用规范标记访问
TCR_ELx.{MTXn, TBIn}    // 启用规范标记属性检查

// 单个 VA 范围
SCTLR_ELx.ATA           // 启用规范标记访问
TCR_ELx.{MTX, TBI}      // 启用规范标记属性检查

    写入规范标记区域：产生阶段 1 权限故障（ESR_ELx.TnD = 1），防止软件篡改规范标记。

    4.6 性能考量¶

    4.6.1 故障模式选择¶

    模式 性能影响 安全性 适用场景
    0b00（忽略） 最低 无检测 兼容性测试
    0b01（同步） 较高 最高（精确报告） 调试、开发
    0b10（异步） 中等 高 生产环境推荐
    0b11（非对称） 中等 写低/读高 写密集型工作负载

    4.6.2 性能优化建议¶

    1. TCMA 免除零标记：对初始化为零的内存区域，启用 TCR_ELx.TCMA0 = 1 免除全零标记的检查
    2. 仅存储检查：对读密集型工作负载，启用 FEAT_MTE_STORE_ONLY 仅对写操作检查
    3. 避免频繁标记切换：使用 ADDG/SUBG 保持标记一致性，减少 LDG/STG 调用
    4. 缓存友好分配：确保分配粒度对齐到标记粒度（16 字节），减少跨标记粒度访问

    5.4 常见漏洞检测场景¶

    漏洞类型 MTE 检测机制 示例
    堆缓冲区溢出 分配的缓冲区具有标记 A，越界区域具有标记 B，使用标记 A 指针访问标记 B 区域触发故障 malloc(32) 后访问第 33 字节
    栈缓冲区溢出 栈帧的局部变量具有不同标记，溢出时跨越标记粒度触发故障 strcpy 越界写入
    释放后使用（UAF） 释放时清除标记为 0，使用旧指针（带非零标记）访问触发故障 free(p) 后通过 p 读写
    双重释放 第二次释放时标记已为 0，分配器检测到异常 free(p); free(p);
    全局缓冲区溢出 全局变量之间使用不同标记，越界访问触发故障 数组越界读写
    内核-用户空间混淆 规范标记模式防止用户空间指针在内核空间使用 用户态指针传入内核
