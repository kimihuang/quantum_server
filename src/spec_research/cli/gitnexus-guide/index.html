    GitNexus CLI Guide¶

    Version: 1.6.3 | 本地代码仓库知识图谱工具 & MCP Server

    GitNexus 是一个本地 CLI 工具和 MCP（Model Context Protocol）服务器，它将代码仓库解析为知识图谱，提供符号级代码导航、执行流分析、变更影响评估、跨仓库依赖追踪等能力。

    快速开始¶

    # 1. 分析（索引）一个仓库
gitnexus analyze /path/to/your/repo

# 2. 查询执行流
gitnexus query "用户登录流程"

# 3. 查看符号上下文
gitnexus context AuthService

# 4. 分析变更影响
gitnexus impact UserService

    核心概念¶

    概念 说明
    知识图谱 GitNexus 将代码解析为符号（函数、类、变量等）及其关系（调用、依赖等）构成的图
    索引 (Index) 对仓库执行 analyze 后生成的 .gitnexus/ 目录，包含知识图谱数据
    全局注册表 ~/.gitnexus/registry.json，记录所有已索引仓库的元信息
    MCP Server 通过 stdio 协议为 AI 编辑器（Cursor、Claude Code 等）提供代码知识接口
    执行流 (Process) 一组按调用关系串联的符号，表示一个完整的业务流程或数据流
    仓库组 (Group) 多个仓库的集合，支持跨仓库的契约追踪和影响分析

    命令 分类 说明
    setup 配置 一次性配置 MCP（Cursor / Claude Code / OpenCode / Codex）

    index 仓库管理 注册已有 .gitnexus/ 目录

    status 仓库管理 查看当前仓库索引状态
    clean 仓库管理 清除当前仓库索引

    query 知识图谱 搜索与概念相关的执行流
    context 知识图谱 符号 360 度视图（调用者、被调用者、所属流程）
    impact 知识图谱 影响半径分析：修改某个符号会影响什么
    cypher 知识图谱 执行原生 Cypher 查询

    仓库管理¶

    analyze — 索引仓库¶

    对仓库进行完整分析，构建知识图谱索引。

    gitnexus analyze [path] [options]

    选项：

    选项 说明
    -f, --force 强制重新索引，即使已是最新
    --embeddings 启用嵌入向量生成（语义搜索），默认关闭
    --drop-embeddings 重建时丢弃已有嵌入
    --skills 从检测到的社区生成仓库特定 skill 文件
    --skip-agents-md 跳过更新 AGENTS.md / CLAUDE.md 中的 gitnexus 部分
    --no-stats 不在 AGENTS.md / CLAUDE.md 中写入文件/符号计数
    --skip-git 对非 Git 目录也进行索引
    --name <alias> 注册时使用自定义别名（区分同名目录）
    --allow-duplicate-name 允许重复别名
    -v, --verbose 启用详细输出
    --max-file-size <kb> 跳过大于此大小的文件（默认 512 KB，上限 32768 KB）

    示例：

    # 基本索引
gitnexus analyze ./my-project

# 强制重建并启用语义搜索
gitnexus analyze ./my-project --force --embeddings

# 使用自定义别名注册
gitnexus analyze ./backend --name "api-server"

# 索引非 Git 项目
gitnexus analyze ./codebase --skip-git

    index — 注册已有索引¶

    将已有的 .gitnexus/ 目录注册到全局注册表，不重新分析。

    gitnexus index [path...] [options]
    选项 说明
    -f, --force 即使 meta.json 缺失也注册
    --allow-non-git 允许注册非 Git 目录

    list — 列出已索引仓库¶

    gitnexus list

    status — 查看索引状态¶

    显示当前仓库的索引状态（需在仓库目录内执行）。

    gitnexus status

    clean — 清除当前仓库索引¶

    删除当前仓库的 .gitnexus/ 目录。

    gitnexus clean [options]
    选项 说明
    -f, --force 跳过确认提示
    --all 清除所有已索引仓库

    remove — 删除指定仓库索引¶

    按别名、名称或绝对路径删除仓库索引，无需在仓库目录内。

    gitnexus remove <target> [options]
    选项 说明
    -f, --force 跳过确认提示 
    # 按别名删除
gitnexus remove api-server

# 按路径删除
gitnexus remove /home/user/projects/backend

    知识图谱查询¶

    query — 搜索执行流¶

    搜索与某个概念相关的执行流。

    gitnexus query <search_query> [options]
    选项 说明
    -r, --repo <name> 目标仓库（仅一个仓库时可省略）
    -c, --context <text> 任务上下文，改善排名
    -g, --goal <text> 你想找什么
    -l, --limit <n> 最大返回执行流数量（默认 5）
    --content 包含完整符号源码

    示例：

    # 搜索登录流程
gitnexus query "用户登录流程"

# 带上下文搜索
gitnexus query "payment" -c "refactoring subscription billing"

# 限制结果数量并包含源码
gitnexus query "error handling" -l 3 --content

    context — 符号 360 度视图¶

    查看一个代码符号的完整上下文：调用者、被调用者、所属流程。

    gitnexus context [name] [options]
    选项 说明
    -r, --repo <name> 目标仓库
    -u, --uid <uid> 直接用符号 UID 查询（无歧义）
    -f, --file <path> 文件路径，用于消除同名歧义
    --content 包含完整符号源码

    示例：

    # 查看 AuthService 的上下文
gitnexus context AuthService

# 用 UID 精确查找
gitnexus context -u "fn:src/auth/service.ts:AuthService:login"

# 按文件消除歧义
gitnexus context Handler -f src/http/middleware.ts

    impact — 影响半径分析¶

    分析修改某个符号会影响哪些其他符号。

    gitnexus impact <target> [options]
    选项 说明
    -d, --direction <dir> 方向：upstream（依赖者）或 downstream（被依赖项），默认 upstream
    -r, --repo <name> 目标仓库
    --depth <n> 最大关系深度（默认 3）
    --include-tests 在结果中包含测试文件

    示例：

    # 查看 UserService 的上游影响（谁依赖它）
gitnexus impact UserService

# 查看下游依赖（它依赖什么），深度 5
gitnexus impact UserService -d downstream --depth 5

# 包含测试文件
gitnexus impact DatabasePool --include-tests

    cypher — 原生 Cypher 查询¶

    对知识图谱执行原始 Cypher 查询语句。

    gitnexus cypher <query> [options]
    选项 说明
    -r, --repo <name> 目标仓库

    示例：

    # 查找所有名为 "Handler" 的符号
gitnexus cypher "MATCH (s:Symbol {name: 'Handler'}) RETURN s"

# 查找某个文件中的所有函数
gitnexus cypher "MATCH (s:Function) WHERE s.file ENDS WITH 'service.ts' RETURN s.name, s.file"

    augment — 增强搜索模式¶

    用知识图谱上下文增强搜索模式（通常用于 hooks）。

    gitnexus augment <pattern>

    变更检测¶

    detect-changes — Git Diff 映射¶

    将 git diff 的变更片段映射到已索引的符号和受影响的执行流。

    gitnexus detect-changes [options]
    选项 说明
    -s, --scope <scope> 分析范围：unstaged、staged、all、compare（默认 unstaged）
    -b, --base-ref <ref> compare 模式的基础分支/提交（如 main）
    -r, --repo <name> 目标仓库

    示例：

    # 查看未暂存的变更影响了哪些符号
gitnexus detect-changes

# 查看相对于 main 分支的变更
gitnexus detect-changes -s compare -b main

# 查看已暂存的变更
gitnexus detect-changes -s staged

    服务模式¶

    serve — Web UI 服务器¶

    启动本地 HTTP 服务器，供 Web UI 连接。

    gitnexus serve [options]
    选项 说明
    -p, --port <port> 端口号（默认 4747）
    --host <host> 绑定地址（默认 127.0.0.1，远程访问用 0.0.0.0）

    示例：

    # 默认启动
gitnexus serve

# 自定义端口，允许远程访问
gitnexus serve -p 8080 --host 0.0.0.0

    mcp — MCP Server¶

    通过 stdio 启动 MCP 服务器，为所有已索引仓库提供服务。

    gitnexus mcp

    通常在编辑器配置（如 claude_desktop_config.json 或 Cursor MCP 设置）中引用此命令。

    eval-server — 评估服务器¶

    启动轻量级 HTTP 服务器，用于评估期间的快速工具调用。

    gitnexus eval-server [options]
    选项 说明
    -p, --port <port> 端口号（默认 4848）
    --idle-timeout <seconds> 空闲自动关闭时间（0 = 禁用，默认 0）

    Wiki 生成¶

    wiki — 生成仓库 Wiki¶

    基于知识图谱，利用 LLM 生成仓库的文档 Wiki。

    gitnexus wiki [path] [options]
    选项 说明
    -f, --force 强制完全重新生成
    --provider <provider> LLM 提供商：openai 或 cursor（默认 openai）
    --model <model> LLM 模型名（默认 minimax/minimax-m2.5）
    --base-url <url> LLM API base URL
    --api-key <key> LLM API Key（保存到 ~/.gitnexus/config.json）
    --api-version <version> Azure api-version 参数
    --reasoning-model 标记为推理模型（o1/o3/o4-mini）
    --no-reasoning-model 禁用推理模型模式
    --concurrency <n> 并行 LLM 调用数（默认 3）
    --gist 生成后发布为 GitHub Gist
    -v, --verbose 显示详细输出（含 LLM 请求/响应）
    --review 在分组后暂停，审查模块结构再生成页面

    示例：

    # 使用 OpenAI 生成 Wiki
gitnexus wiki ./my-project --provider openai --model gpt-4o --api-key sk-xxx

# 使用 Cursor 提供商
gitnexus wiki ./my-project --provider cursor

# 生成后发布为 Gist
gitnexus wiki ./my-project --gist

# 先审查模块结构
gitnexus wiki ./my-project --review

    仓库组（跨仓库分析）¶

    仓库组（Group）用于管理多个相关仓库，支持跨仓库的契约追踪和影响分析。

    group create¶

    创建新的仓库组，生成 group.yaml 模板文件。

    gitnexus group create <name> [options]
    选项 说明
    --force 覆盖已存在的组

    group add / remove¶

    添加或移除组中的仓库。

    # 添加仓库
gitnexus group add <group> <groupPath> <registryName>

# 移除仓库
gitnexus group remove <group> <path>
    • <groupPath>: 层级路径，如 hr/hiring/backend
    • <registryName>: 全局注册表中的仓库名称

    group list / status¶

    列出所有组或查看某个组的详情/状态。

    gitnexus group list [name]
gitnexus group status <name>

    group sync¶

    同步契约注册表 — 提取契约并构建跨仓库链接。

    gitnexus group sync <name> [options]
    选项 说明
    --skip-embeddings 仅使用精确匹配 + BM25
    --exact-only 仅精确匹配
    --allow-stale 跳过过期索引警告
    --verbose 显示每个跨链接详情
    --json JSON 输出

    group impact¶

    跨仓库影响分析。

    gitnexus group impact <name> [options]
    选项 说明
    --target <symbol> 要分析的符号或文件名
    --repo <groupPath> 成员路径（如 app/backend）
    --direction <dir> upstream 或 downstream（默认 upstream）
    --service <path> monorepo 服务目录前缀
    --subgroup <path> 限制参与跨仓库扇出的组
    --max-depth <n> 最大图遍历深度
    --cross-depth <n> 跨仓库跳数
    --min-confidence <n> 最小关系置信度（0-1）
    --include-tests 包含测试文件
    --timeout-ms <n> 本地影响分析超时（毫秒）
    --json JSON 输出

    group query¶

    跨仓库搜索执行流。

    gitnexus group query <name> <query> [options]
    选项 说明
    --subgroup <path> 限制搜索范围
    --limit <n> 最大合并结果数（默认 5）
    --json JSON 输出

    group contracts¶

    查看契约注册表。

    gitnexus group contracts <name> [options]
    选项 说明
    --type <type> 按契约类型筛选
    --repo <repo> 按仓库筛选
    --unmatched 仅显示未匹配的契约
    --json JSON 输出

    初始配置¶

    setup — 一次性配置 MCP¶

    为 Cursor、Claude Code、OpenCode、Codex 配置 MCP 连接。

    gitnexus setup

    该命令会自动检测已安装的编辑器并写入相应的 MCP 配置。

    配置文件与环境变量¶

    配置文件：

    文件 说明
    ~/.gitnexus/registry.json 全局仓库注册表
    ~/.gitnexus/config.json 全局配置（API Key 等）
    .gitnexus/ 仓库索引目录（每个仓库内）
    .gitnexusignore 索引忽略规则（支持 .gitignore 语法及否定模式）
    group.yaml 仓库组定义文件

    环境变量：

    变量 说明
    GITNEXUS_NO_GITIGNORE=1 跳过 .gitignore 解析（仍读取 .gitnexusignore）
    GITNEXUS_MAX_FILE_SIZE=N 覆盖大文件跳过阈值（KB），默认 512，最大 32768

    .gitnexusignore 示例：

    # 忽略构建产物
dist/
build/
*.min.js

# 取消忽略（即使默认被过滤也强制包含）
!__tests__/
!src/**/*.test.ts
