    1. 概述¶

    Repo 是 Google 为管理多个 Git 仓库而开发的工具，广泛用于 Android、Chromium OS 等大型项目中。它与 Gerrit Code Review 系统紧密集成，主要功能包括：

    • 多仓库管理: 通过清单文件（Manifest）定义和同步数百个 Git 仓库
    • 代码审查集成: 与 Gerrit 无缝对接，支持上传、下载、cherry-pick 变更
    • 批量操作: 在所有或选定的项目上并行执行命令
    • 分支管理: 跨项目的 topic 分支创建、切换、变基和清理
    • 多清单支持: 支持嵌套清单和子清单树结构

    工作区结构¶

    <workspace>/
├── .repo/                          # Repo 内部数据目录
│   ├── manifests/                  # 清单仓库（包含 default.xml 等）
│   │   ├── default.xml             # 默认清单文件
│   │   └── ...
│   ├── manifest.xml                # 当前使用的清单软链接
│   ├── local_manifests/            # 本地清单（用户自定义）
│   ├── project-objects/            # 共享对象存储
│   ├── projects/                   # 项目 git 数据
│   ├── repo/                       # repo 工具自身
│   └── ...
├── <project-1>/                    # 项目 1 的检出目录
├── <project-2>/                    # 项目 2 的检出目录
└── ...

    2. 安装与更新¶

    安装¶

    # 方法 1: 通过包管理器（Ubuntu/Debian）
sudo apt install repo

# 方法 2: 手动安装（推荐）
mkdir -p ~/.bin
curl https://storage.googleapis.com/git-repo-downloads/repo > ~/.bin/repo
chmod a+x ~/.bin/repo
export PATH=~/.bin:$PATH

# 方法 3: 使用国内镜像
curl https://mirrors.tuna.tsinghua.edu.cn/git/git-repo > ~/.bin/repo
chmod a+x ~/.bin/repo

    更新¶

    # 手动更新
repo selfupdate

# repo sync 时自动更新（默认行为）

    使用国内镜像加速¶

    # 设置环境变量使用清华镜像
export REPO_URL='https://mirrors.tuna.tsinghua.edu.cn/git/git-repo'

    3. 全局选项¶

    repo [-p|--paginate|--no-pager] [--color=COLOR] [--trace] [--time] [--version] COMMAND [ARGS]
    选项 说明
    -h, --help 显示帮助信息
    --help-all 显示所有子命令的帮助信息
    -p, --paginate 使用分页器显示输出
    --no-pager 禁用分页器
    --color=COLOR 控制颜色输出: auto, always, never
    --trace 跟踪 git 命令执行（等同于 REPO_TRACE=1）
    --trace-to-stderr 跟踪输出同时发送到 stderr 和 .repo/TRACE_FILE
    --trace-python 跟踪 Python 命令执行
    --time 计时 repo 命令执行时间
    --version 显示 repo 版本
    --show-toplevel 显示工作区顶级目录路径
    --event-log=FILE 事件日志文件路径
    --git-trace2-event-log=DIR Git trace2 事件日志目录
    --submanifest-path=REL_PATH 子清单路径

    4. 核心工作流¶

    4.1 初始化与同步¶

    # 1. 初始化工作区
mkdir my-project && cd my-project
repo init -u https://github.com/myorg/manifest -b main

# 2. 同步所有项目
repo sync

# 3. （可选）同步到已知良好版本
repo smartsync

    4.2 日常开发流程¶

    # 1. 创建开发分支
repo start my-feature --all

# 2. 修改代码...

# 3. 查看状态
repo status

# 4. 暂存文件
repo stage -i

# 5. 提交代码
git commit -am "feat: add new feature"

# 6. 变基到最新
repo rebase

# 7. 上传审查
repo upload

# 8. 审查通过后清理分支
repo prune

    4.3 代码审查流程¶

    # 下载他人的变更到本地
repo download project-name 12345/2

# 审查后挑选变更到另一个分支
repo cherry-pick <sha1>

# 上传自己的变更
repo upload --re=reviewer@example.com --cc=team@example.com

    5. 命令详解¶

    5.1 init - 初始化客户端¶

    在当前目录初始化一个 repo 客户端。

    repo init [options] [manifest url]

    清单选项:

    选项 说明
    -u URL, --manifest-url=URL 清单仓库地址（也可作为第一个位置参数）
    -b REVISION, --manifest-branch=REVISION 清单分支或修订版本（使用 HEAD 表示默认分支）
    --manifest-upstream-branch=BRANCH 当 -b 指定为 commit 时，指定包含该 commit 的 ref 名称
    -m NAME.xml, --manifest-name=NAME.xml 初始清单文件名（默认 default.xml）
    -g GROUP, --groups=GROUP 限制同步的项目组，如 default\|all\|G1,G2,G3\|G4,-G5
    -p PLATFORM, --platform=PLATFORM 限制平台组: auto\|all\|none\|linux\|darwin\|...
    --submodules 同步清单仓库的子模块
    --standalone-manifest 下载清单为静态文件而非 git 仓库
    --manifest-depth=DEPTH 清单仓库浅克隆深度（0 为完整克隆，默认 0）

    清单检出选项:

    选项 说明
    -c, --current-branch 仅获取当前清单分支（默认）
    --no-current-branch 获取所有清单分支
    --tags 获取清单标签
    --no-tags 不获取清单标签

    检出模式:

    选项 说明
    --mirror 创建远程仓库的镜像副本而非工作目录
    --archive 以归档方式检出每个项目（见 git archive）
    --worktree 使用 git-worktree 管理项目

    项目检出优化:

    选项 说明
    --reference=DIR 引用镜像目录（加速克隆）
    --dissociate 克隆后与引用镜像断开关联
    --depth=DEPTH 浅克隆深度（见 git clone）
    --partial-clone 启用部分克隆（默认 clone-filter 为 blob:none）
    --no-partial-clone 禁用部分克隆
    --partial-clone-exclude=PROJECTS 排除指定项目不使用部分克隆
    --clone-filter=CLONE_FILTER 部分克隆的过滤选项（默认 blob:none）
    --clone-bundle 启用 HTTP/HTTPS 的 /clone.bundle（默认）
    --no-clone-bundle 禁用 clone.bundle
    --git-lfs 启用 Git LFS 支持
    --no-git-lfs 禁用 Git LFS 支持

    Repo 版本选项:

    选项 说明
    --repo-url=URL Repo 仓库地址（$REPO_URL）
    --repo-rev=REV Repo 分支或修订版本（$REPO_REV）
    --no-repo-verify 不验证 repo 源代码

    其他选项:

    选项 说明
    --config-name 总是提示输入 name/email

    示例:

    # 基本初始化
repo init -u https://android.googlesource.com/platform/manifest -b main

# 使用浅克隆加速
repo init -u https://github.com/myorg/manifest -b main --depth=1

# 引用本地镜像加速
repo init -u https://github.com/myorg/manifest --reference /path/to/mirror

# 使用部分克隆节省空间
repo init -u https://github.com/myorg/manifest --partial-clone

# 使用静态清单文件
repo init -u https://example.com/manifest.xml --standalone-manifest

# 同步指定组的项目
repo init -u https://github.com/myorg/manifest -g all

# 切换清单分支（在已有客户端中）
repo init -b release-branch
repo sync

    5.2 sync - 同步项目¶

    同步本地项目目录与清单中指定的远程仓库。

    repo sync [<project>...]

    核心选项:

    选项 说明
    -j JOBS, --jobs=JOBS 并行任务数（默认: 基于 CPU 核数）
    --jobs-network=JOBS 网络并行任务数（需配合 --no-interleaved）
    --jobs-checkout=JOBS 本地检出并行任务数（需配合 --no-interleaved）
    --fail-fast 第一个错误时立即停止
    -f, --force-sync 强制覆盖指向不同对象目录的 git 目录（可能丢失数据）
    --force-checkout 强制检出，即使有未提交修改（可能丢失数据）
    --force-remove-dirty 强制移除清单中已不存在的脏项目（可能丢失数据）
    --rebase 变基本地提交，无论是否已发布
    -l, --local-only 仅更新工作树，不抓取
    --no-manifest-update, --nmu 使用现有清单，不更新到最新修订
    -n, --network-only 仅抓取，不更新工作树
    -d, --detach 将项目分离回清单修订版本
    -c, --current-branch 仅获取当前分支
    --no-current-branch 获取所有分支

    智能同步:

    选项 说明
    -s, --smart-sync 使用清单服务器的最新已知良好构建进行同步
    -t SMART_TAG, --smart-tag=SMART_TAG 使用指定标签的已知良好构建同步
    -u USERNAME, --manifest-server-username 清单服务器认证用户名
    -p PASSWORD, --manifest-server-password 清单服务器认证密码

    高级选项:

    选项 说明
    --interleaved 并行执行 fetch 和 checkout（默认）
    --no-interleaved 分阶段执行 fetch 和 checkout
    -m NAME.xml, --manifest-name=NAME.xml 临时使用指定清单
    --fetch-submodules 从服务器获取子模块
    --tags 获取标签
    --no-tags 不获取标签（默认）
    --optimized-fetch 仅获取本地不存在的 SHA1 固定项目
    --retry-fetches=N 瞬态错误时的重试次数
    --prune 删除远程已不存在的 ref（默认）
    --no-prune 不删除远程已不存在的 ref
    --auto-gc 在所有同步项目上运行垃圾回收
    --no-auto-gc 不运行垃圾回收（默认）

    Hook 选项:

    选项 说明
    --no-verify 不运行 post-sync hook
    --verify 不提示直接运行 post-sync hook
    --ignore-hooks post-sync hook 失败时不中止

    SSH 连接特性:

    当项目远程 URL 使用 SSH 连接（ssh://, git+ssh://, user@host:path）时，repo 会自动启用 SSH ControlMaster 选项，允许多个项目复用同一 SSH 隧道。

    要禁用此功能: export GIT_SSH=ssh

    示例:

    # 同步所有项目
repo sync

# 并行同步（8 个任务）
repo sync -j8

# 同步指定项目
repo sync platform/system/core build/soong

# 强制同步（覆盖本地变更）
repo sync -f -j8

# 仅获取不检出
repo sync -n

# 分离回清单版本
repo sync -d

# 使用智能同步
repo sync -s

# 第一个失败时停止
repo sync --fail-fast

# 智能同步并指定认证
repo sync -s -u myuser -p mypassword

    5.3 start - 创建分支¶

    从清单指定的修订版本创建新的开发分支。

    repo start <newbranchname> [--all | <project>...]
    选项 说明
    -j JOBS, --jobs=JOBS 并行任务数
    --all 在所有项目中创建分支
    -r REVISION, --rev=REVISION 将分支指向指定修订版本而非 upstream
    --head, --HEAD --rev HEAD 的缩写

    示例:

    # 在所有项目中创建分支
repo start my-feature --all

# 在指定项目中创建分支
repo start my-feature platform/system/core build/soong

# 从 HEAD 创建分支
repo start hotfix --all --HEAD

    5.4 upload - 上传代码审查¶

    将变更发送到 Gerrit Code Review 系统。

    repo upload [--re --cc] [<project>]...

    核心选项:

    选项 说明
    -j JOBS, --jobs=JOBS 并行任务数
    -t, --topic-branch 将 topic 设置为本地分支名
    --topic=TOPIC 设置变更的 topic
    --hashtag=HASHTAGS, --ht=HASHTAGS 添加 hashtag（逗号分隔）
    --hashtag-branch, --htb 将本地分支名添加为 hashtag
    -l LABELS, --label=LABELS 上传时添加标签
    --pd=DESC, --patchset-description=DESC 补丁集描述
    --re=REVIEWERS, --reviewers=REVIEWERS 请求审查者
    --cc=CC 抄送邮箱地址
    --br=BRANCH, --branch=BRANCH 上传的本地分支
    -c, --current-branch 上传当前 git 分支
    --no-current-branch 上传所有 git 分支
    -D BRANCH, --dest=BRANCH 提交审查的目标分支
    -n, --dry-run 试运行（不实际上传）
    -y, --yes 自动回答 yes

    状态控制:

    选项 说明
    -w, --wip 上传为 Work-in-Progress 变更
    -r, --ready 标记为 Ready（清除 WIP 状态）
    -ne, --no-emails 上传时不发送邮件

    安全选项:

    选项 说明
    --no-cert-checks 禁用 SSL 证书验证（不安全）
    --ignore-untracked-files 忽略工作目录中的未跟踪文件

    示例:

    # 上传当前分支
repo upload -c

# 上传并指定审查者和抄送
repo upload --re=reviewer@company.com --cc=team@company.com

# 上传为 WIP
repo upload -w -c

# 标记为 Ready
repo upload -r -c

# 试运行
repo upload -n -c

# 上传到指定目标分支
repo upload -c -D release-branch

# 添加 hashtag
repo upload -c --ht "feature,jira-1234"

# 自动确认上传
repo upload -c -y

    5.5 download - 下载变更¶

    从审查系统下载变更到本地工作目录。

    repo download {[project] change[/patchset]}...
    选项 说明
    -b BRANCH, --branch=BRANCH 先创建新分支
    -c, --cherry-pick 使用 cherry-pick 而非 checkout
    -x, --record-origin cherry-pick 时传递 -x 参数
    -r, --revert 使用 revert 而非 checkout
    -f, --ff-only 强制快进合并

    示例:

    # 下载 Gerrit 变更 12345 的 patchset 2
repo download 12345/2

# 下载指定项目的变更
repo download platform/system/core 12345/2

# 下载到新分支
repo download -b my-review 12345/2

# 使用 cherry-pick 方式
repo download -c 12345/2

    5.6 abandon - 放弃分支¶

    永久删除开发分支（等同于 git branch -D）。

    repo abandon [--all | <branchname>] [<project>...]
    选项 说明
    -j JOBS, --jobs=JOBS 并行任务数
    --all 删除所有项目中的所有分支

    示例:

    # 删除指定分支
repo abandon my-feature

# 删除指定项目中的分支
repo abandon my-feature platform/system/core

# 删除所有项目中的所有分支
repo abandon --all

    5.7 branches / branch - 查看分支¶

    查看当前可用的 topic 分支。branch 和 branches 是同一命令的别名。

    repo branches [<project>...]

    输出格式说明:

     *P nocolor                   | in repo
    repo2                     |
    列 说明
    第 1 列 * 表示该分支在某个项目中被检出，空格表示没有
    第 2 列 空: 未发布 / P: 所有 commit 已发布 / p: 部分 commit 已发布
    第 3 列 分支名称
    第 4 列 (| 后) 该分支出现的项目列表（未列出则表示在所有项目中）

    5.8 checkout - 切换分支¶

    检出由 repo start 创建的已有分支。

    repo checkout <branchname> [<project>...]

    等同于: repo forall [<project>...] -c git checkout <branchname>

    示例:

    # 切换到指定分支
repo checkout my-feature

# 在指定项目中切换
repo checkout my-feature platform/system/core

    5.9 rebase - 变基操作¶

    将本地 topic 分支的变更变基到 upstream HEAD。

    repo rebase {[<project>...] | -i <project>...}
    选项 说明
    --fail-fast 第一个错误时停止
    -f, --force-rebase 传递 --force-rebase 给 git rebase
    --no-ff 传递 --no-ff 给 git rebase
    --autosquash 传递 --autosquash 给 git rebase
    --whitespace=WS 传递 --whitespace 给 git rebase
    --auto-stash 变基前 stash 本地修改
    -m, --onto-manifest 变基到清单版本而非 upstream HEAD
    -i, --interactive 交互式变基（仅限单个项目）

    示例:

    # 在所有项目中变基
repo rebase

# 变基前自动 stash
repo rebase --auto-stash

# 交互式变基（单个项目）
repo rebase -i platform/system/core

# 变基到清单版本
repo rebase -m

    5.14 forall - 批量执行命令¶

    在每个项目中执行相同的 shell 命令。

    repo forall [<project>...] -c <command> [<arg>...]
repo forall -r str1 [str2] ... -c <command> [<arg>...]
    选项 说明
    -c, --command 要执行的命令（必选）
    -r, --regex 仅在匹配正则/通配符的项目上执行
    -i, --inverse-regex 仅在不匹配的项目上执行
    -g GROUPS, --groups=GROUPS 仅在指定组的项目上执行
    -e, --abort-on-errors 命令执行失败时中止
    --ignore-missing 忽略缺失的检出，不报错
    --interactive 强制交互模式（等同 -j1）
    -p 在输出前显示项目标题（适合搜索）
    -j JOBS, --jobs=JOBS 并行任务数

    可用环境变量:

    变量 说明
    REPO_PROJECT 项目唯一名称
    REPO_PATH 相对于客户端根目录的路径
    REPO_OUTERPATH 子清单根目录相对于客户端根目录的路径
    REPO_INNERPATH 相对于子清单根目录的路径
    REPO_REMOTE 清单中定义的远程名称
    REPO_LREV 清单修订版本翻译为本地跟踪分支名
    REPO_RREV 清单中原始修订版本字符串
    REPO_COUNT 总项目数
    REPO_I 当前迭代计数（从 1 开始）
    REPO__* 项目 annotation 元素中定义的自定义变量

    pwd 设为项目工作目录。$1, $2, ..., $# 设为 <command> 之后的参数。

    示例:

    # 列出所有项目名
repo forall -c 'echo $REPO_PROJECT'

# 查看所有项目的当前分支
repo forall -c 'git branch --show-current'

# 在所有项目中搜索某个字符串
repo forall -p -c 'git log -SFoo'

# 仅在匹配的项目中执行
repo forall -r '.*system.*' -c 'git status'

# 仅在指定组中执行
repo forall -g darwin -c 'echo $REPO_PATH'

# 使用进度指示
repo forall -c 'echo "[$REPO_I/$REPO_COUNT] $REPO_PROJECT"'

# 使用项目 annotation 中的变量
repo forall -c 'echo $REPO__MY_ANNOTATION'

    5.15 grep - 跨项目搜索¶

    在所有项目文件中搜索指定模式。

    repo grep {pattern | -e pattern} [<project>...]

    搜索源:

    选项 说明
    --cached 搜索索引而非工作树
    -r TREEish, --revision=TREEish 搜索指定 TREEish 而非工作树

    模式选项:

    选项 说明
    -e PATTERN 搜索模式
    -i, --ignore-case 忽略大小写
    -a, --text 将二进制文件作为文本处理
    -I 不在二进制文件中匹配
    -w, --word-regexp 仅在单词边界匹配
    -v, --invert-match 选择不匹配的行
    -G, --basic-regexp POSIX 基本正则表达式（默认）
    -E, --extended-regexp POSIX 扩展正则表达式
    -F, --fixed-strings 固定字符串匹配（非正则）

    模式组合:

    选项 说明
    --all-match 仅匹配包含所有模式的行
    --and, --or, --not 布尔运算符组合模式
    -(, -) 布尔运算符分组

    输出选项:

    选项 说明
    -n 显示匹配行的行号
    -C CONTEXT 显示匹配行上下文
    -B CONTEXT 显示匹配行前方行
    -A CONTEXT 显示匹配行后方行
    -l, --name-only 仅显示文件名
    -L, --files-without-match 仅显示不匹配的文件名

    示例:

    # 简单搜索
repo grep "CONFIG_UART"

# 忽略大小写
repo grep -i "config_uart"

# 搜索包含 #define 和 MAX_PATH 或 PATH_MAX 的行
repo grep -e '#define' --and -\( -e MAX_PATH -e PATH_MAX \)

# 仅显示文件名
repo grep -l "TODO"

# 搜索指定项目
repo grep "main" build/soong

# 搜索特定 TREEish
repo grep -r HEAD~5 "deprecated"

    5.16 list - 列出项目¶

    5.17 manifest - 清单检查¶

    6. Manifest 清单文件格式¶

    清单文件是 XML 格式，描述了 repo 客户端的结构。清单保存在一个 Git 仓库中，通常为 default.xml。

    6.1 基本结构¶

    <?xml version="1.0" encoding="UTF-8"?>
<manifest>
  <notice>可选的通知文本</notice>

  <remote name="origin" fetch="https://github.com/myorg" />

  <default remote="origin" revision="main" sync-j="4" />

  <manifest-server url="https://manifest-server.example.com" />

  <project name="platform/system/core" path="system/core" />
  <project name="platform/build" path="build" groups="notdefault" />

  <include name="extra.xml" />
</manifest>

    6.2 元素详解¶

    <remote> - 远程仓库¶

    <remote name="origin"
        alias="upstream"
        fetch="https://github.com/myorg"
        pushurl="ssh://git@github.com/myorg"
        review="https://review.example.com/"
        revision="main" />
    属性 必选 说明
    name 是 唯一短名称（用于 .git/config）
    alias 否 覆盖 name 作为实际 remote 名称（可重复）
    fetch 是 URL 前缀，fetch + / + project.name + .git 组成完整 URL
    pushurl 否 push URL 前缀
    review 否 Gerrit 审查服务器地址
    revision 否 该远程的默认分支

    <default> - 默认值¶

    <default remote="origin"
         revision="main"
         dest-branch="main"
         upstream="main"
         sync-j="4"
         sync-j-max="32"
         sync-c="true"
         sync-s="true"
         sync-tags="false" />
    属性 说明
    remote 未指定 remote 的项目使用此远程
    revision 未指定 revision 的项目使用此分支
    dest-branch repo upload 的目标分支
    upstream SHA1 所在的 ref 名称（用于锁定版本清单的 -c 模式同步）
    sync-j 同步时的并行任务数
    sync-j-max 最大并行任务数
    sync-c true 表示仅同步指定分支
    sync-s true 表示也同步子项目
    sync-tags false 表示不同步标签

    <project> - 项目¶

    <project name="platform/system/core"
         path="system/core"
         remote="origin"
         revision="main"
         dest-branch="main"
         groups="default,linux"
         upstream="main"
         clone-depth="1"
         sync-c="true"
         sync-s="false"
         sync-tags="true"
         force-path="true">
  <annotation name="my-annotation" value="my-value" />
  <copyfile src="LICENSE" dest="TOP_LEVEL_LICENSE" />
  <linkfile src="build.sh" dest="build.sh" />
</project>
    属性 说明
    name 唯一名称（必选），用于构建 URL: ${remote_fetch}/${name}.git
    path 本地检出路径（默认与 name 相同）
    remote 使用的远程（默认为 <default> 中的设置）
    revision 跟踪的分支/标签/SHA
    dest-branch repo upload 目标分支
    groups 所属项目组（默认属于 all, name:<name>, path:<path>）
    clone-depth 克隆深度（覆盖 --depth 选项）
    upstream SHA1 所在 ref
    sync-c 是否仅同步指定分支
    sync-s 是否同步子项目
    force-path 使用 path 而非 name 创建本地镜像

    子元素:

    元素 说明
    <annotation> 名称-值对，在 forall 中可通过 REPO__<name> 访问
    <copyfile> 同步时将 src 文件复制到 dest（相对于工作区顶层）
    <linkfile> 同步时创建从 dest 到 src 的符号链接

    <extend-project> - 扩展项目¶

    修改已有项目的属性，常用于本地清单。

    <extend-project name="platform/system/core"
                revision="my-branch"
                groups="default,my-group" />

    <remove-project> - 移除项目¶

    从内部清单表中删除项目。

    <remove-project name="platform/system/core" />
<remove-project path="system/core" optional="true" />

    <include> - 包含清单¶

    包含另一个清单文件。

    <include name="extra.xml" groups="group1" revision="main" />

    <manifest-server> - 清单服务器¶

    用于智能同步的 XML RPC 服务地址。

    <manifest-server url="https://manifest-server.example.com" />

    <submanifest> - 子清单¶

    <submanifest name="chromeos"
             project="chromiumos/manifest"
             manifest-name="default.xml"
             revision="main"
             path="src"
             groups="chromeos"
             default-groups="chromeos,default" />

    <repo-hooks> - Repo 钩子¶

    <repo-hooks in-project="tools/repo"
            enabled-list="pre-upload,post-upload" />

    <superproject> - 超级项目¶

    <superproject name="platform/superproject" remote="origin" revision="main" />

    <contactinfo> - 联系信息¶

    <contactinfo bugurl="https://bugs.example.com" />

    6.3 完整示例¶

    <?xml version="1.0" encoding="UTF-8"?>
<manifest>

  <notice>
    This manifest is for internal development only.
  </notice>

  <remote name="origin"
          fetch="https://github.com/myorg"
          review="https://review.example.com/" />

  <remote name="upstream"
          fetch="https://github.com/upstream-org" />

  <default remote="origin"
           revision="main"
           dest-branch="main"
           sync-j="8" />

  <manifest-server url="https://manifest-server.example.com" />

  <!-- 核心项目 -->
  <project name="kernel" path="kernel" groups="linux" />
  <project name="platform/system/core" path="system/core" />
  <project name="platform/build/soong" path="build/soong" />

  <!-- 可选项目 -->
  <project name="vendor/hal/nordic" path="vendor/hal/nordic" groups="bluetooth,notdefault" />
  <project name="vendor/hal/stm32" path="vendor/hal/stm32" groups="stm32,notdefault" />

  <!-- 上游项目 -->
  <project name="cmsis" path="external/cmsis" remote="upstream" revision="v5.9.0" />

  <!-- 带文件操作的项目 -->
  <project name="build/make" path="build/make">
    <copyfile src="core/root.mk" dest="root.mk" />
    <linkfile src="envsetup.sh" dest="envsetup.sh" />
    <annotation name="maintainer" value="build-team@example.com" />
  </project>

  <!-- 包含额外清单 -->
  <include name="extras/default.xml" groups="extras" />

</manifest>

    7. 多清单管理 (Multi-manifest)¶

    大多数 repo 命令支持以下多清单选项:

    选项 说明
    --outer-manifest 从最外层清单开始操作
    --no-outer-manifest 不在外部清单上操作
    --this-manifest-only 仅在当前（子）清单上操作
    --no-this-manifest-only, --all-manifests 在当前清单及其子清单上操作

    使用 --submanifest-path=REL_PATH 全局选项可指定子清单路径。

    8. 本地清单 (Local Manifests)¶

    本地清单允许用户添加额外的项目，存放在 $TOP_DIR/.repo/local_manifests/*.xml。

    # 创建本地清单目录
mkdir -p .repo/local_manifests

# 创建本地清单
cat > .repo/local_manifests/local.xml << 'EOF'
<?xml version="1.0" encoding="UTF-8"?>
<manifest>
  <!-- 覆盖上游项目的修订 -->
  <extend-project name="platform/system/core" revision="my-feature" />

  <!-- 添加自定义项目 -->
  <project name="my/custom/project" path="my-project" />

  <!-- 移除不需要的项目 -->
  <remove-project name="vendor/unused" optional="true" />
</manifest>
EOF

# 同步以应用本地清单
repo sync

    本地清单中的项目会被自动添加到 local::<filename> 组。

    9. Gerrit 集成与 upload 配置¶

    9.1 上传自动确认¶

    在 .git/config 中设置:

    [review "https://review.example.com/"]
    autoupload = true

    或:

    git config --global review.https://review.example.com/.autoupload true

    9.2 自动审查者和抄送¶

    [review "https://review.example.com/"]
    autoreviewer = reviewer1@example.com,reviewer2@example.com
    autocopy = team@example.com

    9.3 自动 Topic¶

    [review "https://review.example.com/"]
    uploadtopic = true

    9.4 自动 Hashtag¶

    [review "https://review.example.com/"]
    uploadhashtags = feature,jira-123

    9.5 上传标签¶

    [review "https://review.example.com/"]
    uploadlabels = Code-Review+1,Verified+1

    9.6 上传警告阈值¶

    # 超过 10 个 commit 时警告
git config --global review.https://review.example.com/.uploadwarningthreshold 10

    10. 常见场景与最佳实践¶

    10.1 镜像管理¶

    # 创建镜像
repo init -u https://github.com/myorg/manifest --mirror
repo sync

# 从镜像初始化客户端（加速）
repo init -u /path/to/mirror --reference /path/to/mirror
repo sync

# 使用镜像后断开关联
repo init -u https://github.com/myorg/manifest --reference /path/to/mirror --dissociate

    10.2 大型项目同步优化¶

    # 部分克隆节省磁盘空间
repo init -u https://github.com/myorg/manifest --partial-clone

# 浅克隆
repo init -u https://github.com/myorg/manifest --depth=1

# 高并行同步
repo sync -j32

# 仅获取不检出（CI 缓存）
repo sync -n -j32

# 使用 clone.bundle 加速
repo init -u https://github.com/myorg/manifest
repo sync --clone-bundle -j16

    10.3 代码审查工作流¶

    # 1. 创建分支
repo start feature-x --all

# 2. 修改并提交...

# 3. 变基到最新
repo rebase

# 4. 上传审查
repo upload -c -w --re=reviewer@example.com

# 5. 根据反馈修改...

# 6. 重新上传（标记为 Ready）
repo upload -c -r

# 7. 合并后清理
repo prune

    10.4 多人协作分支¶

    # 查看所有未合并的分支
repo overview

# 查看某人的工作
repo info -d -o --current-branch

# 批量变基所有项目
repo rebase --fail-fast

    # 同步后验证状态

    11. Repo 与 West 对比¶

    特性 Repo West
    清单格式 XML YAML
    主要用途 Android/大型项目 Zephyr RTOS
    代码审查 Gerrit 集成 无内置
    配置系统 Git 配置风格 Git 配置风格
    扩展机制 Repo hooks Python 扩展命令
    项目组 groups 属性 groups + group-filter
    清单导入 <include> import 字段
    镜像支持 --mirror 模式 无内置
    智能同步 manifest-server 无
    分支管理 start/checkout/abandon/prune 无内置
    upload/download Gerrit 原生支持 无内置
    本地清单 .repo/local_manifests/ 无
