    1. 概述¶

    West 是 Zephyr RTOS 的元工具（meta-tool），用于管理由多个 Git 仓库组成的工作区。它类似于 Android 的 repo 工具，主要功能包括：

    • 工作区初始化: 克隆清单仓库，建立多仓库工作区
    • 项目管理: 根据清单文件克隆、更新多个 Git 仓库
    • 批量操作: 在所有项目上执行 git 命令
    • 配置管理: Git 风格的三级配置系统
    • 扩展机制: 支持项目自定义扩展命令

    工作区结构¶

    <workspace>/
├── .west/                    # West 内部数据目录
│   └── config                # 本地配置文件
├── <manifest-project>/       # 清单仓库（包含 west.yml）
│   └── west.yml              # 主清单文件
├── <project-1>/              # 清单中定义的项目 1
├── <project-2>/              # 清单中定义的项目 2
└── ...

    2. 安装¶

    pip install west

    依赖项: colorama, packaging, pykwalify, PyYAML

    验证安装:

    west --version
# West version: v1.5.0

    3. 全局选项¶

    west [-h] [-z ZEPHYR_BASE] [-v] [-q] [-V] <command> ...
    选项 说明
    -h, --help 获取帮助信息
    -z, --zephyr-base 覆盖 Zephyr 基础目录，默认为清单中路径为 "zephyr" 的项目
    -v, --verbose 显示详细输出，可多次使用以增加详细程度
    -q, --quiet 减少输出详细程度，可多次使用
    -V, --version 打印版本号并退出

    4. 核心工作流¶

    4.1 从远程仓库初始化¶

    # 使用默认的 Zephyr 仓库初始化
west init <workspace>

# 指定清单仓库 URL
west init -m https://github.com/my-org/manifest-repo <workspace>

# 指定清单仓库 URL 和分支
west init -m https://github.com/my-org/manifest-repo --mr v2.0 <workspace>

# 使用浅克隆加速
west init -m https://github.com/my-org/manifest-repo -o --depth=1 <workspace>

# 切换到工作区并更新所有项目
cd <workspace>
west update

    4.2 从本地仓库初始化¶

    # 基于已有的本地清单仓库创建工作区
west init -l /path/to/local/manifest-repo

cd /path/to/local/manifest-repo
west update

    4.3 典型日常操作¶

    # 查看所有项目状态
west status

# 更新所有项目
west update

# 更新指定项目
west update hal_drivers cmsis

# 查看项目列表
west list

# 在所有项目中搜索
west grep "CONFIG_UART"

    5. 命令详解¶

    5.1 init - 初始化工作区¶

    创建一个新的 west 工作区。

    # 用法 1: 从远程克隆清单仓库
west init [-m URL] [--mr REVISION] [--mf FILE] [-o GIT_CLONE_OPT] [directory]

# 用法 2: 使用本地清单仓库
west init -l [--mf FILE] directory

    参数:

    参数 说明
    directory 使用 -l 时为本地清单仓库路径；不使用 -l 时为要创建的工作区目录（默认为当前目录）

    选项:

    选项 说明
    -m, --manifest-url 清单仓库 URL，不能与 -l 同时使用
    --mr, --manifest-rev 清单仓库的分支或标签名，默认使用远程默认分支
    --mf, --manifest-file 清单文件名（默认为 west.yml）
    -l, --local 将 directory 作为已有的本地清单仓库，.west 目录创建在其旁边
    -o, --clone-opt 传递给 git clone 的额外选项（如 -o=--depth=1），可多次使用
    --rename-delay 重命名临时目录前的等待秒数（NTFS 文件系统兼容性选项）

    示例:

    # 使用自定义清单仓库初始化
west init -m https://github.com/myorg/my-manifest --mr main my-workspace

# 使用本地清单仓库
west init -l ~/repos/my-manifest

# 浅克隆 + 自定义清单文件
west init -m https://github.com/myorg/manifest --mf manifest-custom.yml -o=--depth=1 my-workspace

    5.2 update - 更新项目¶

    根据清单文件更新活动项目。

    west update [PROJECT ...]

    参数:

    参数 说明
    PROJECT 要操作的项目（按名称或路径），默认为所有活动项目

    选项:

    选项 说明
    --stats 打印更新操作的性能统计信息

    抓取行为:

    选项 说明
    -f, --fetch {always,smart} 抓取策略: always 总是抓取，smart（默认）跳过本地已有的 SHA 或标签
    -o, --fetch-opt 传递给 git fetch 的额外选项（如 o=--depth=1），可多次使用
    -n, --narrow 仅抓取项目修订版本，不传递 --tags

    已检出分支行为:

    选项 说明
    -k, --keep-descendants 如果已检出的分支是新 manifest-rev 的后代，保持检出状态（优先级高于 --rebase）
    -r, --rebase 将已检出的分支变基到新的 manifest-rev 上（错误时保留部分变基结果）

    本地缓存:

    选项 优先级 说明
    --name-cache 0（最高） 缓存目录结构与项目名称匹配的扁平布局
    --path-cache 1 缓存目录结构与工作区路径布局一致
    --auto-cache 2（最低） 自动管理的缓存，每个仓库存储在以 URL 基名命名的目录下

    高级选项:

    选项 说明
    --group-filter, --gf 追加组过滤器，可多次使用
    --submodule-init-config git submodule init 时设置的 git 配置选项，格式为 <option>=<value>，可多次使用

    示例:

    # 更新所有项目
west update

# 仅更新指定项目
west update hal_drivers cmsis

# 强制从远程抓取所有项目的最新代码
west update -f always

# 保持已有分支的变基
west update -r

# 使用本地 name 缓存加速初始化
west update --name-cache /path/to/cache

# 使用自动缓存（首次自动创建，后续自动同步）
west update --auto-cache /path/to/auto-cache

    5.3 list - 列出项目¶

    打印清单中定义的项目信息。

    west list [PROJECT ...]

    选项:

    选项 说明
    -a, --all 包含非活动项目
    -i, --inactive 仅列出非活动项目
    --manifest-path-from-yaml 打印清单文件 YAML 中定义的清单仓库路径
    -f, --format FORMAT 自定义格式字符串

    格式字符串变量:

    变量 说明
    {name} 项目名称
    {description} 项目描述
    {url} 完整的远程 URL
    {path} 相对于顶级目录的路径
    {abspath} 绝对路径
    {posixpath} POSIX 风格的绝对路径（使用 / 分隔符）
    {revision} 清单中定义的修订版本
    {sha} 修订版本对应的 SHA（需要项目已克隆）
    {cloned} cloned 或 not-cloned
    {active} active 或 inactive
    {clone_depth} 克隆深度，未指定则为 None
    {groups} 项目所属组，逗号分隔

    默认格式: {name:12} {path:28} {revision:40} {url}

    示例:

    # 列出所有活动项目
west list

# 列出所有项目（含非活动的）
west list -a

# 仅列出非活动项目
west list -i

# 自定义输出格式
west list -f "{name:20} {sha:12} {path}"

# 查看指定项目
west list hal_drivers cmsis

# 列出所有已克隆项目
west list -f "{name} {cloned}" -a

    5.4 manifest - 管理清单¶

    管理 west 清单文件。

    west manifest (--resolve | --freeze | --validate | --path | --untracked)

    操作（必须且只能指定一个）:

    操作 说明
    --resolve 解析所有 import 后打印等效的单个清单文件（需要导入的清单已通过 west update 克隆）
    --freeze 类似 --resolve，但将所有项目修订版本转换为当前 SHA（需要所有项目已克隆）
    --validate 验证清单文件是否能成功解析，成功则无输出并正常退出
    --path 打印顶级清单文件的路径
    --untracked 打印工作区中不被 west 管理或跟踪的所有文件和目录（类似 git status 的未跟踪文件）

    --resolve 和 --freeze 的额外选项:

    选项 说明
    -o, --out OUT 输出到文件，默认为标准输出
    --active-only 仅解析活动项目

    示例:

    # 验证清单文件
west manifest --validate

# 查看清单文件路径
west manifest --path

# 解析所有导入，输出合并后的清单
west manifest --resolve

# 解析并输出到文件
west manifest --resolve -o resolved.yml

# 冻结所有修订为 SHA
west manifest --freeze -o locked.yml

# 查看未跟踪的文件
west manifest --untracked

    5.5 compare - 对比状态¶

    将每个项目的当前状态与最近一次 west update 的结果进行对比。

    west compare [PROJECT ...]

    对每个项目，以下任一条件为真时会输出信息:

    1. HEAD 与 manifest-rev 分支指向的提交不同
    2. 工作目录有未提交的更改
    3. 有本地检出的分支（可通过配置或 --ignore-branches 禁用此检查）
    选项 说明
    -a, --all 包含非活动项目
    --exit-code 如果有任何项目输出了状态信息，则以状态码 1 退出
    --ignore-branches 对检出的分支且工作目录干净、分支位于 west update 相同提交的项目跳过输出
    --no-ignore-branches 覆盖 --ignore-branches 或 compare.ignore-branches 配置选项

    示例:

    # 查看所有项目与清单的差异
west compare

# 包含非活动项目
west compare -a

# 在 CI 中使用退出码判断
west compare --exit-code

    5.6 diff - 项目差异¶

    在指定项目上运行 git diff。

    west diff [PROJECT ...]

    未识别的参数会直接传递给 git diff。

    选项 说明
    -a, --all 包含非活动项目
    -m, --manifest 显示相对于 manifest-rev 的更改

    示例:

    # 查看所有项目的差异
west diff

# 查看指定项目的差异
west diff hal_drivers

# 相对于 manifest-rev 的差异
west diff -m

# 传递 git diff 参数
west diff --stat hal_drivers

    5.7 status - 项目状态¶

    在指定项目上运行 git status。

    west status [PROJECT ...]

    未识别的参数会直接传递给 git status。

    选项 说明
    -a, --all 包含非活动项目

    注意: 若要查找工作区中未被 west 管理的未跟踪文件，请使用 west manifest --untracked。

    示例:

    # 查看所有项目的 git 状态
west status

# 查看指定项目
west status hal_drivers cmsis

# 包含非活动项目
west status -a

    5.8 forall - 批量执行命令¶

    在每个指定项目的仓库中运行 shell 命令。

    west forall -c COMMAND [PROJECT ...]

    可用环境变量:

    变量 说明
    WEST_PROJECT_NAME 项目名称
    WEST_PROJECT_PATH 项目相对路径
    WEST_PROJECT_ABSPATH 项目绝对路径
    WEST_PROJECT_REVISION 项目修订版本
    WEST_PROJECT_URL 项目远程 URL
    WEST_PROJECT_REMOTE 项目远程名称
    选项 说明
    -c COMMAND 要执行的命令（必选）
    -C CWD 命令执行目录，默认为各项目路径
    -a, --all 包含非活动项目
    -g, --group GROUPS 仅在指定组中的项目上运行命令，可多次使用（任一匹配即运行）

    示例:

    # 查看所有项目的当前分支
west forall -c "git branch --show-current"

# 查看所有项目到最后一次提交的日期
west forall -c 'git log -1 --format="%cd" --date=short'

# 仅在 hal 组的项目中执行
west forall -c "git status" -g hal

# 在所有项目中搜索特定字符串（使用环境变量）
west forall -c "echo \$WEST_PROJECT_NAME: \$(grep -r 'CONFIG_UART' --include='*.h' -l | head -5)"

# 从统一目录执行命令（但环境变量仍指向各项目）
west forall -c "make \$WEST_PROJECT_NAME" -C /path/to/build

    5.9 grep - 跨项目搜索¶

    在一个或多个项目中运行 grep 或类似 grep 的工具。

    west grep [PATTERN]

    选项:

    选项 说明
    --tool {git-grep,ripgrep,grep} 选择搜索工具（默认 git-grep）
    --tool-path PATH 指定工具的可执行文件路径
    -p, --project PROJECT 指定要搜索的项目，可多次使用（默认为所有已克隆的活动项目）

    工具说明:

    工具 说明 默认搜索路径
    git-grep 使用 git grep（默认） 搜索 git
    ripgrep 使用 ripgrep 搜索 rg，然后 ripgrep
    grep 使用系统 grep --recursive 搜索 grep

    颜色控制:

    默认情况下，当 color.ui 配置为 true 时强制彩色输出。可通过命令行覆盖:

    west grep --color=never "pattern"

    或通过配置永久设置:

    west config grep.color never

    示例:

    # 在所有已克隆项目中搜索（使用 git grep）
west grep "CONFIG_UART"

# 使用 ripgrep 搜索
west grep --tool ripgrep "CONFIG_UART"

# 在特定项目中搜索
west grep -p hal_drivers "CONFIG_UART"

# 使用 git grep 的未跟踪文件选项
west grep --untracked "CONFIG_UART"

# 传递额外参数给搜索工具
west grep -- "CONFIG_UART" -- '*.h'

# 持久化更改默认工具
west config grep.tool ripgrep

    5.10 config - 配置管理¶

    West 配置文件管理，遵循 Git 风格的三级配置文件约定。

    west config [name] [value]

    操作（最多指定一个）:

    操作 说明
    （无） 读取配置值
    -l, --list 列出所有配置项及其值
    -d, --delete 删除一个配置项（在设置它的第一个文件中删除）
    -D, --delete-all 在所有配置文件中删除该配置项
    -a, --append 追加到已有值（配置项必须已存在）

    配置文件位置（最多指定一个）:

    选项 说明
    --system 系统级配置
    --global 用户级配置
    --local 当前工作区配置

    未指定时，读取使用所有三个文件（优先级: local > global > system），写入使用 local 文件。

    配置文件默认路径:

    级别 Linux macOS Windows
    系统 /etc/westconfig /usr/local/etc/westconfig %PROGRAMDATA%\west\config
    全局 ~/.westconfig 或 $XDG_CONFIG_HOME/west/config ~/.westconfig 用户主目录下 .westconfig
    本地 <workspace>/.west/config <workspace>/.west/config <workspace>/.west/config

    可通过环境变量 WEST_CONFIG_SYSTEM、WEST_CONFIG_GLOBAL、WEST_CONFIG_LOCAL 覆盖。

    示例:

    # 读取配置值
west config manifest.path

# 设置配置值（写入 local）
west config manifest.path zephyr

# 追加到配置值
west config -a build.cmake-args -- " -DEXTRA_CFLAGS='-Wextra'"

# 追加组过滤器
west config -a manifest.group-filter ,+optional

# 列出所有配置
west config -l

# 删除配置项
west config -d build.cmake-args

# 在全局文件中设置
west config --global color.ui true

    5.11 topdir - 获取顶级目录¶

    打印当前 west 工作区的顶级目录的绝对路径。

    west topdir

    该目录包含 .west 目录，所有项目路径都相对于此目录。

    5.12 help - 帮助¶

    获取 west 或某个命令的帮助信息。

    west help [command_name]

    6. Manifest 清单文件格式¶

    清单文件（默认为 west.yml）是 YAML 格式，定义了工作区中所有项目的信息。

    6.1 顶级结构¶

    manifest:
  version: "1.2"           # 清单架构版本（可选）
  defaults:                # 默认值（可选）
    remote: myremote
    revision: main
  remotes:                 # 远程仓库定义（可选）
    - name: myremote
      url-base: https://github.com/myorg
  projects:                # 项目列表
    - name: project1
      remote: myremote
      revision: v1.0
  group-filter:            # 组过滤器（可选）
    - +group1
    - -group2
  self:                    # 清单仓库自身配置（可选）
    path: manifest-repo

    6.2 字段详解¶

    version（可选）¶

    清单架构最低版本号。当前版本为 1.2。

    defaults（可选）¶

    defaults:
  remote: myremote    # 默认远程名称
  revision: main      # 默认修订版本（不指定时为 master）

    remotes（可选）¶

    remotes:
  - name: myremote          # 远程名称（必选）
    url-base: https://github.com/myorg   # URL 前缀（必选）

    项目的完整 URL = remote.url-base + / + project.repo-path（默认为 project.name）

    projects（可选）¶

    projects:
  - name: project1                # 项目唯一名称（必选）
    description: "项目描述"        # 描述（可选，无实际作用）
    remote: myremote              # 远程名称（不能与 url 同时使用）
    repo-path: custom-path        # 仓库路径（默认与 name 相同）
    url: https://example.com/repo # 完整 URL（不能与 remote 同时使用）
    revision: v1.0                # 分支、标签或 SHA（默认为 master）
    path: my-project              # 本地检出路径（默认与 name 相同）
    submodules: true              # 子模块配置（bool 或列表）
    clone-depth: 1                # 克隆深度限制
    west-commands: commands.yml   # 扩展命令文件路径
    import: true                  # 导入该项目的清单数据
    groups:                       # 项目所属组
      - group1
      - group2
    userdata:                     # 自定义数据
      custom: value

    import 字段详解:

    import 用于从其他项目中导入清单数据，实现清单的模块化管理:

    # 导入所有内容
import: true

# 仅导入指定项目
import:
  - hal_drivers
  - cmsis

# 使用 name-filter 导入
import:
  name-filter:
    - hal_*

# 使用 name-prefix 添加前缀
import:
  name-prefix: myproj_

# 使用 name-only 避免名称冲突
import:
  name-only: true

    self（可选）¶

    清单仓库自身的配置:

    self:
  path: manifest-repo       # 清单仓库的路径
  west-commands: cmd.yml    # 扩展命令文件路径
  import:                   # 从清单仓库导入额外清单
    - extra-manifest.yml
  userdata:                 # 自定义数据
    key: value

    group-filter（可选）¶

    控制哪些项目组被启用或禁用:

    group-filter:
  - +group1       # 启用 group1
  - -group2       # 禁用 group2
  - +group3

    也可通过配置设置:

    west config manifest.group-filter ,-group2,+group1

    manifest:
  version: "1.2"

  defaults:
    remote: myorg
    revision: main

  remotes:
    - name: myorg
      url-base: https://github.com/myorg

      url-base: https://github.com/upstream-org

  self:
    path: workspace-manifest

    7. 配置系统¶

    7.1 三级配置优先级¶

    配置值按以下优先级加载（后者覆盖前者）:

    系统配置 (最低) < 全局配置 < 本地配置 (最高)

    7.2 常用配置项¶

    manifest.path¶

    清单仓库的路径（默认在 west init 时自动设置）。

    west config manifest.path

    manifest.group-filter¶

    组过滤器，控制哪些项目组处于活动状态。

    west config manifest.group-filter ,-optional,+bluetooth

    color.ui¶

    控制彩色输出。

    west config color.ui true

    compare.ignore-branches¶

    west compare 是否忽略已检出分支的输出。

    west config compare.ignore-branches true

    grep.tool¶

    默认的 grep 工具。

    west config grep.tool ripgrep

    grep.color¶

    grep 命令的颜色设置。

    west config grep.color never

    build.cmake-args¶

    CMake 构建参数（追加模式示例）:

    west config -a build.cmake-args -- " -DEXTRA_CFLAGS='-Wextra'"

    8. 扩展命令¶

    West 支持通过 west-commands.yml 文件扩展命令。项目可以在清单中声明扩展命令:

    # west-commands.yml
west-commands:
  - file: scripts/my_commands.py
    commands:
      - name: build
        class: Build
        help: "Build the project"
      - name: flash
        class: Flash
        help: "Flash the board"

    然后在 west.yml 中引用:

    projects:
  - name: my_project
    west-commands: scripts/west-commands.yml

    或用于清单仓库自身:

    self:
  west-commands: scripts/west-commands.yml

    扩展命令通过继承 WestCommand 类实现:

    from west.commands import WestCommand

class Build(WestCommand):
    def __init__(self):
        super().__init__(
            'build',
            'Build the project',
            'Builds the firmware image.'
        )

    def do_add_parser(self, parser_adder):
        parser = parser_adder.add_parser(self.name, help=self.help)
        parser.add_argument('-b', '--board', required=True)
        return parser

    def do_run(self, args, unknown_args):
        print(f"Building for board: {args.board}")

    10. 最佳实践¶

    10.1 工作区管理¶

    # 初始化新工作区后，先验证清单
west init -m <url> workspace && cd workspace
west manifest --validate

# 更新前查看将要发生的变化
west update -n    # -n 使用窄抓取，减少数据量

# 大型团队协作中使用本地缓存
west update --auto-cache /shared/cache

    10.2 锁定版本¶

    # 生成锁定版本的清单（所有修订转为 SHA）
west manifest --freeze -o locked-manifest.yml

# 后续可使用锁定清单进行精确复现

    10.3 项目组管理¶

    # 按需启用/禁用功能模块
west config manifest.group-filter ,-wifi,+bluetooth,+usb

# 仅更新特定组的项目
west update --gf +bluetooth

    10.4 CI/CD 集成¶

    # 验证工作区状态（适合 CI 检查）
west compare --exit-code

# 检查未提交的更改
west diff

# 使用浅克隆加速 CI
west update -o=--depth=1 -f always

    10.5 跨项目操作¶

    # 在所有项目中统计代码行数
west forall -c "git ls-files | xargs wc -l | tail -1"

# 在所有项目中清理未跟踪文件
west forall -c "git clean -fd"

# 批量拉取最新代码（不使用 west update）
west forall -c "git pull --rebase"
