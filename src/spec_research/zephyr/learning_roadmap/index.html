    Zephyr RTOS 学习路线图¶

    环境参考¶

    项目 值
    Zephyr 版本 v4.4.99 (main)
    目标板 qemu_cortex_m3 (ARM Cortex-M3)
    Workspace /home/lion/workdir/sourcecode/quantum_main/src/zephyrproject
    编译命令 west build -p -b qemu_cortex_m3 -d build_xxx zephyr/samples/<path>
    运行命令 west build -d build_xxx -t run

    学习原则¶

    1. 先跑通再理解 — 每个概念先编译运行示例，再看源码
    2. 边改边学 — 修改示例代码观察行为变化，而非只阅读
    3. QEMU 为主 — 前期全部在 QEMU 上完成，避免硬件依赖阻塞进度
    4. 看官方文档为辅 — Zephyr 文档非常完整，遇到概念不清时查阅 zephyr/doc/ 或 https://docs.zephyrproject.org

    第一阶段：基础入门（1-2 周）¶

    目标： 理解 Zephyr 的项目结构、编译流程和最基本的应用程序。

    1.1 编译系统与项目结构¶

    学习内容： - West workspace 结构（.west/, zephyr/, modules/, build/） - CMake 构建流程 - Kconfig 配置系统 - Device Tree（设备树）基础

    实践： - [ ] 阅读已有 build/ 目录结构，理解 zephyr.dts, .config, CMakeLists.txt 的作用 - [ ] 修改 samples/hello_world 的 prj.conf，启用 CONFIG_PRINTK=y，观察差异 - [ ] 修改 prj.conf 中的 CONFIG_SYS_CLOCK_TICKS_PER_SEC，观察 philosophers 示例的行为变化 - [ ] 查看 build/zephyr/.config，搜索与 hello_world 相关的配置项

    关键文件： - zephyr/samples/hello_world/prj.conf — 项目配置 - zephyr/boards/qemu/cortex_m3/qemu_cortex_m3.dts — 设备树 - zephyr/boards/qemu/cortex_m3/qemu_cortex_m3_defconfig — 默认配置 - build/zephyr/zephyr.dts — 编译生成的完整设备树

    1.2 最小应用与启动流程¶

    学习内容： - Zephyr 启动流程（_start -> z_cstart -> main()） - main() 函数的返回值处理（return 0 vs k_sleep(K_FOREVER)）

    实践： - [ ] 编译运行 samples/basic/minimal，阅读 src/main.c - [ ] 修改 minimal 为打印一条消息，编译运行验证 - [ ] 对比 hello_world 和 minimal 的 .config 差异：diff build_hello/zephyr/.config build_minimal/zephyr/.config

    1.3 printk 与日志¶

    学习内容： - printk() — 基础打印（无缓冲，同步） - LOG_*() 系列 — 分级日志（INFO, WRN, ERR, DBG） - 日志后端配置

    实践： - [ ] 在 hello_world 中使用不同日志级别 - [ ] 修改 prj.conf 启用 CONFIG_LOG=y 和 CONFIG_LOG_DEFAULT_LEVEL=4 - [ ] 使用 LOG_INF(), LOG_WRN(), LOG_ERR() 替代 printk()

    第二阶段：内核核心（2-3 周）¶

    目标： 掌握 Zephyr 的线程模型、调度、同步原语。

    2.1 线程（Thread）¶

    学习内容： - 线程创建：K_THREAD_DEFINE() 宏 - 线程优先级（数值越小优先级越高） - 抢占式线程 vs 协作式线程 - 线程延迟与睡眠：k_sleep(), k_msleep(), k_busy_wait()

    实践： - [ ] 阅读并运行 samples/synchronization，理解两个线程并发执行 - [ ] 创建 3 个不同优先级的线程，观察调度顺序 - [ ] 用协作式线程（优先级为负数）验证不会被抢占

    关键 API： 

    K_THREAD_DEFINE(name, stack_size, entry, p1, p2, p3, prio, options, delay);
k_thread_start(&name);
k_yield();
k_sleep(K_MSEC(100));

    2.2 同步原语¶

    学习内容：

    原语 API 用途
    Semaphore k_sem_init(), k_sem_give(), k_sem_take() 计数信号量
    Mutex k_mutex_init(), k_mutex_lock(), k_mutex_unlock() 互斥锁（支持优先级继承）
    Condition Variable k_condvar_init(), k_condvar_wait(), k_condvar_signal() 条件变量

    实践： - [ ] 阅读运行 samples/philosophers，分析互斥锁使用方式 - [ ] 阅读运行 samples/kernel/condition_variables/condvar - [ ] 编写一个生产者-消费者程序：1 个线程生产数据放入环形缓冲区，1 个线程消费

    2.3 线程间通信¶

    学习内容：

    机制 API 特点
    Message Queue k_msgq_init(), k_msgq_put(), k_msgq_get() 固定大小消息，支持紧急消息
    FIFO k_fifo_init(), k_fifo_put(), k_fifo_get() 链表式，可传递不同大小数据
    LIFO k_lifo_init(), k_lifo_put(), k_lifo_get() 栈式，后进先出
    Pipe k_pipe_init(), k_pipe_put(), k_pipe_get() 字节流管道

    实践： - [ ] 阅读运行 samples/kernel/msg_queue，理解普通消息和紧急消息 - [ ] 实现：3 个发送线程向消息队列发送数据，1 个接收线程按序处理 - [ ] 用 FIFO 实现相同功能，体会与消息队列的区别

    2.4 定时器与时间管理¶

    学习内容： - k_timer_init(), k_timer_start(), k_timer_stop() — 内核定时器 - k_uptime_get() — 系统运行时间 - k_cycle_get_32() — 获取 CPU 周期数 - Timeout 机制（与 k_sem_take 等配合的超时等待）

    实践： - [ ] 实现一个周期性定时器，每 500ms 触发一次回调 - [ ] 实现带超时的信号量等待：k_sem_take(&sem, K_MSEC(1000))

    第三阶段：设备与驱动（2-3 周）¶

    目标： 理解 Zephyr 的设备模型和驱动框架。

    3.1 设备模型¶

    学习内容： - Device Tree 节点与驱动绑定机制 - DEVICE_DT_DEFINE() 宏 - const struct device * 设备实例 - devicetree.h — DT 宏 API 读取设备树信息

    实践： - [ ] 阅读 samples/basic/custom_dts_binding，理解自定义 DT 绑定 - [ ] 阅读 qemu_cortex_m3 的设备树，找到 UART 节点，追踪到驱动代码 - [ ] 修改设备树添加一个虚拟节点，在代码中用 DT_NODELABEL() 读取其属性

    3.2 GPIO 驱动¶

    学习内容： - gpio_dt_spec 结构体 - gpio_pin_configure_dt(), gpio_pin_toggle_dt() - GPIO 中断回调

    实践： - [ ] 阅读 samples/drivers/gpio（需要带 GPIO 的 QEMU 板或硬件） - [ ] 尝试在 qemu_x86 上编译运行 GPIO 示例（qemu_x86 有 GPIO 支持）

    3.3 UART / Console¶

    学习内容： - UART 设备 API - Console 子系统 - shell 子系统（命令行接口）

    实践： - [ ] 编译运行 samples/subsys/shell/shell_module - [ ] 注册一个自定义 Shell 命令，交互式调用 - [ ] 理解 CONFIG_UART_CONSOLE 和 CONFIG_CONSOLE 的区别

    第四阶段：子系统与高级特性（3-4 周）¶

    目标： 掌握 Zephyr 的常用子系统，能开发完整应用。

    4.1 文件系统¶

    学习内容： - Zephyr File System API（fs_open(), fs_read(), fs_write()） - 支持的后端：FAT, LittleFS, NVS

    实践： - [ ] 阅读 samples/subsys/fs - [ ] 在 QEMU 中运行 FS 示例（使用 RAM disk 后端）

    4.2 网络协议栈¶

    学习内容： - Zephyr 网络架构概述 - Socket API（BSD 兼容） - DHCP, DNS, MQTT, CoAP 等协议

    实践： - [ ] 阅读 samples/net/ethernet（qemu_cortex_m3 不支持，可换 qemu_x86） - [ ] 阅读 samples/net/mqtt_publisher

    4.3 电源管理¶

    学习内容： - 低功耗状态（PM_DEVICE_STATE_SUSPEND, PM_DEVICE_STATE_OFF） - 系统电源策略 - Tickless idle

    实践： - [ ] 阅读 samples/subsys/pm - [ ] 对比开启/关闭 CONFIG_PM=y 后的功耗差异

    4.4 存储¶

    学习内容： - Flash API（flash_read(), flash_write(), flash_erase()） - NVS（Non-Volatile Storage） - Settings 子系统（键值存储）

    实践： - [ ] 阅读 samples/drivers/flash_shell - [ ] 阅读 samples/subsys/settings

    第五阶段：项目实战（持续）¶

    目标： 综合运用所学知识，开发实际项目。

    5.1 项目建议（按难度递增）¶

    难度 项目 涉及知识
    初级 温度监控报警系统 传感器驱动 + 定时器 + 日志
    中级 简易 Shell 管理工具 Shell 子系统 + 文件系统 + 命令注册
    中级 MQTT 数据上报客户端 网络 + MQTT + JSON 序列化
    高级 OTA 固件升级 Flash + Bootloader + 网络
    高级 蓝牙传感器节点 Bluetooth + 传感器 + 低功耗

    5.2 自定义 Board 定义¶

    学习内容： - Board YAML 文件结构 - 自定义 Device Tree overlay - Board specific Kconfig

    实践： - [ ] 在 zephyr/boards/qemu/ 下创建自定义 board，修改内存布局和设备 - [ ] 使用 -D 参数传入 overlay：west build -Doverlay.conf

    5.3 模块开发¶

    学习内容： - Zephyr Module 结构（zephyr/module.yml） - 在 west manifest 中引用自定义模块 - 可复用驱动/库的模块化封装

    推荐学习资源¶

    资源 说明
    Zephyr 官方文档 最权威的参考，包含所有 API 文档
    zephyr/doc/ 本地文档源码，可离线查看
    zephyr/samples/ 大量示例代码，是最好的学习材料
    zephyr/include/zephyr/kernel.h 内核 API 头文件
    zephyr/tests/ 内核功能测试用例，展示 API 边界用法
    Zephyr 源码阅读 深入理解实现细节

    源码阅读顺序¶

    建议按以下顺序阅读 Zephyr 源码，由浅入深：

    1. samples/hello_world/src/main.c          — 最简应用入口
2. zephyr/kernel/init.c                    — 系统启动流程
3. zephyr/kernel/thread.c                  — 线程管理核心
4. zephyr/kernel/sched.c                   — 调度器
5. zephyr/kernel/sem.c                     — 信号量实现
6. zephyr/include/zephyr/kernel.h          — 内核 API 声明
7. zephyr/drivers/serial/uart_stellaris.c  — UART 驱动（qemu_cortex_m3 使用）
8. zephyr/include/zephyr/device.h          — 设备模型
9. zephyr/include/zephyr/devicetree.h      — 设备树 API

    常用调试技巧¶

    # 查看完整的 Kconfig 配置
cat build/zephyr/.config | grep CONFIG_ | less

# 查看编译后的设备树
cat build/zephyr/zephyr.dts | less

# 查看内存使用详情
cat build/zephyr/zephyr.map | grep -A5 "Memory region"

# 修改 Kconfig 配置（菜单式配置）
west build -t menuconfig

# 增加编译详细度
west build -v

# 只编译不链接（快速验证语法）
west build --target zephyr/include/generated/zephyr/autoconf.h
