    Zephyr 启动过程分析¶

    基于 zephyr.map 文件和源代码分析，本文档详细描述了 Zephyr RTOS 的完整启动流程。

    目标平台¶

    • 架构: ARM Cortex-M (ARMv7-M)
    • MCU: TI LM3S6965
    • 工具链: GNU ARM-Zephyr-EABI GCC 14.3.0
    • C 库: Picolibc

    详细启动阶段分析¶

    阶段 1: 复位向量 (Reset Vector)¶

    入口点: z_arm_reset @ 0x00000634

    源代码位置: zephyr/arch/arm/core/cortex_m/reset.S

    执行流程:

    复位 → __start → z_arm_reset

    关键操作:

    步骤 地址偏移 操作 说明
    1 0x634 设置 MSP MSP = z_main_stack + CONFIG_MAIN_STACK_SIZE
    2 - 清零 BSS (调试用) 如果启用 CONFIG_DEBUG_THREAD_INFO
    3 - 调用 soc_reset_hook() SoC 特定复位钩子
    4 - 初始化架构硬件 调用 z_arm_init_arch_hw_at_boot()
    5 - 锁定中断 cpsid i 或设置 BASEPRI
    6 - 初始化看门狗 如果启用 CONFIG_WDOG_INIT
    7 - 清零中断堆栈 如果启用 CONFIG_INIT_STACKS
    8 - 设置 PSP PSP = z_interrupt_stacks + ISR_STACK_SIZE
    9 0x6DE 跳转到 z_prep_c() bl z_prep_c

    关键符号 (来自 map 文件):

    .text._reset_section
               0x00000634       0x34 zephyr/arch/.../reset.S.obj
               0x00000634                z_arm_reset
               0x00000634                __start

    阶段 2: C 环境准备 (z_prep_c)¶

    函数地址: z_prep_c @ 0x00001a7a

    源代码位置: zephyr/arch/arm/core/cortex_m/prep_c.c

    关键操作序列:

    void z_prep_c(void)
{
    soc_prep_hook();                           // SoC 准备钩子
    relocate_vector_table();                    // 向量表重定位
    z_arm_floating_point_init();               // FPU 初始化 (如果有)
    arch_bss_zero();                           // BSS 段清零
    arch_data_copy();                          // 数据段复制
    z_arm_interrupt_init();                    // 中断控制器初始化
    arch_cache_init();                         // 缓存初始化
    z_cstart();                                // 跳转到内核启动
}

    向量表重定位 (relocate_vector_table) @ 0x0000068c:

    .text.relocate_vector_table
               0x0000068c       0x20 zephyr/.../prep_c.c.obj
               0x0000068c                relocate_vector_table

    BSS 段清零和数据分析复制:

    函数 地址 功能
    arch_bss_zero 编译器提供 将 .bss 段清零
    arch_data_copy 编译器提供 XIP 模式下复制 .data 段

    阶段 3: 内核启动 (z_cstart)¶

    函数地址: z_cstart @ 0x00000c54

    源代码位置: zephyr/kernel/init.c

    初始化级别顺序:

    INIT_LEVEL_EARLY
    │
    ▼
INIT_LEVEL_PRE_KERNEL_1
    │
    ▼
INIT_LEVEL_PRE_KERNEL_2

    详细代码流程:

    void z_cstart(void)
{
    gcov_static_init();                        // Gcov 覆盖率初始化
    z_sys_init_run_level(INIT_LEVEL_EARLY);    // EARLY 级别初始化

    arch_kernel_init();                        // 架构内核初始化
    LOG_CORE_INIT();                           // 日志核心初始化

    z_dummy_thread_init(&_thread_dummy);        // 初始化 dummy 线程
    z_device_state_init();                     // 设备状态初始化

    soc_early_init_hook();                     // SoC 早期初始化
    board_early_init_hook();                   // 板级早期初始化

    // 基础硬件初始化
    z_sys_init_run_level(INIT_LEVEL_PRE_KERNEL_1);  // UART、Console 等
    z_sys_init_run_level(INIT_LEVEL_PRE_KERNEL_2); // 系统时钟等

    // 准备多线程环境
    switch_to_main_thread(prepare_multithreading());
}

    阶段 7: 后内核初始化 (bg_thread_main)¶

    函数地址: bg_thread_main @ 0x00000b30

    源代码位置: zephyr/kernel/init.c

    执行流程:

    static void bg_thread_main(void *unused1, void *unused2, void *unused3)
{
    z_mem_manage_init();                        // 内存管理初始化
    z_sys_post_kernel = true;                   // 标记后内核阶段

    z_sys_init_run_level(INIT_LEVEL_POST_KERNEL);
    soc_late_init_hook();
    board_late_init_hook();

    z_sys_init_run_level(INIT_LEVEL_APPLICATION);

    z_init_static_threads();                    // 静态线程初始化

    // 调用应用程序 main 函数
    main();
}

    启动时间线¶

    时间点 0:     硬件复位
     │
     ├─ 1. z_arm_reset (汇编)
     │     - 初始化 MSP
     │     - 调用 soc_reset_hook()
     │     - 锁定中断
     │
     ├─ 2. z_prep_c (C)
     │     - relocate_vector_table()
     │     - arch_bss_zero()
     │     - arch_data_copy()
     │     - z_arm_interrupt_init()
     │
     ├─ 3. z_cstart (C)
     │     - INIT_LEVEL_EARLY
     │     - arch_kernel_init()
     │     - INIT_LEVEL_PRE_KERNEL_1
     │     │     └─ uart_stellaris_init()
     │     │     └─ uart_console_init()
     │     - INIT_LEVEL_PRE_KERNEL_2
     │     │     └─ sys_clock_driver_init()
     │     - prepare_multithreading()
     │     - switch_to_main_thread()
     │
     ├─ 4. bg_thread_main (C) - 运行在主线程
     │     - INIT_LEVEL_POST_KERNEL
     │     │     └─ malloc_init()
     │     - INIT_LEVEL_APPLICATION
     │     - main()
     │
     └─ 5. main() - 应用程序入口

    总结¶

    Zephyr 的启动过程遵循严格的阶段化初始化模型:

    1. 硬件复位阶段: 最小化的汇编代码，设置基本运行环境
    2. C 环境准备: 清零 BSS、复制数据、初始化中断控制器
    3. 内核早期初始化: 日志、设备驱动早期初始化
    4. PRE_KERNEL_1: UART、SoC 特定硬件
    5. PRE_KERNEL_2: 系统时钟
    6. 线程准备: 调度器、线程创建
    7. 上下文切换: 从启动代码切换到主线程
    8. POST_KERNEL: 内存管理、文件系统等
    9. APPLICATION: 应用层初始化
    10. main(): 用户应用程序入口
