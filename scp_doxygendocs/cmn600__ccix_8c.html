CMN-600 CCIX 功能实现

实现 CMN-600 的 CCIX (Cache Coherent Interconnect for Accelerators) 配置功能，包括：

  • CCIX Gateway 链路建立序列
  • RAID (Requesting Agent ID) / LDID (Logical Device ID) 映射编程
  • AgentID-to-LinkID 映射编程
  • CXRA SAM 地址区域配置
  • SMP 模式使能
  • 协议信用交换
  • 系统一致性域和 DVM 域进入
  • CCIX 能力信息获取

CCIX 网关配置设置

完整的 CCIX Gateway 配置流程：

  1. 如需 SMP 模式则使能
  2. 为本地 RN-F/RN-D/RN-I 代理分配 RAID 并编程 LDID-to-RAID 映射
  3. 为所有本地代理编程 AgentID-to-LinkID 映射
  4. 配置 CXHA HA ID
  5. 为远端 RN-F 代理编程 CXHA RAID-to-LDID 映射和 HN-F LDID-to-CHI-Node-ID
  6. 配置 CXRA SAM 地址区域
  7. 启动 CCIX 链路 UP 序列

使能并启动 CCIX 链路 UP 序列

执行完整的 CCIX 链路建立流程：

  1. 配置 CXLA 协议参数（优化 TLP、消息打包、最大包大小、PCIe 总线号、Traffic Class、Vendor ID）
  2. 设置链路使能位，等待 EN 位置位
  3. 验证链路 DOWN 状态（UP 位清零、DOWN 位置位、ACK 位清零）
  4. 发起链路 UP 请求（REQ 位置位），等待 ACK 位置位和 DOWN 位清零
参数
[in]ctxCMN-600 模块上下文指针
[in]config远端节点 CCIX 配置数据
[in]link_idCCIX 链路标识符（0-2）
返回
FWK_SUCCESS 成功；FWK_E_PARAM link_id 超出范围； 其他错误码表示链路建立超时

static

使能 SMP (Symmetric Multiprocessing) 模式

根据 CMN-600 修订版本采用不同的 SMP 模式配置方式：

  • r2p0：通过 CXRA/CXHA UNIT_INFO 的只读位判断硬件是否支持
  • r3p0+：通过 CXRA/CXHA/CXLA AUX_CTRL 的可写位配置使能
  • r1p3 及以下：无 SMP 模式寄存器，直接返回成功
参数
[in]ctxCMN-600 模块上下文指针
返回
FWK_SUCCESS 成功；FWK_E_SUPPORT r2p0 版本硬件不支持 SMP 模式
